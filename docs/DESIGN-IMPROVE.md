## P0 — PDF 문서 (치명, 최우선)
1. **글리프 깨짐 수정**: NanumGothic CID 서브셋 매핑 파손 — 폰트 임베드 방식 교체(서브셋 끄기 또는 Pretendard/NotoSansKR 정품 TTF 재확보 후 전체 임베드). **검증 기준 변경: pdftoppm으로 이미지 렌더 → Read 눈검수 필수**(텍스트 추출만으론 불충분 — 이번 사고의 교훈)
2. **문서 리디자인** (4종: 확인서·명세서·안전리포트·계약서): 청구·법적 문서 격식 —
   상단 헤더(문서명 크게·발행처 상호·발행일), 확인서는 종이 시안 미학(오렌지 포인트 절제 사용·구분선), 표는 명확한 괘선·우측정렬 금액·합계 강조 박스, **서명란: 서명 이미지 + 서명자명 + 일시를 도장 느낌 박스로**, 하단 "작업온에서 발행" 소형 푸터. 여백 규칙 통일

## P1 — 앱 UX (직관성)
- "(SIGNED)" 등 영문 enum 노출 전량 한국어 라벨화
- 서류 묶음 보내기: 롱프레스 은닉 → **눈에 보이는 선택 모드 버튼**
- 상세·정산 화면 죽은 공간 제거: 핵심 숫자·다음 행동 버튼을 위로
- 장부 버튼 과잉 정리: 주 행동 1개(입금 기록) + 나머지는 ⋯ 메뉴로
- D-day 부호·표기 통일(수금 D-3 / 만료 D-7 / 지남 +N일 규칙 하나로)
- 아이콘 스타일 일관화(스트로크 2px 계열로 통일)
- 홈: 오늘 할 일이 없어도 "확인서 쓰기" 큰 CTA가 항상 명확히

## P2 — 웹·공개 페이지
- 종이 확인서(웹 /c)·앱 상세: **서명 이미지 실제 표시**(현재 텍스트만 — 시그니처 미달)

- 죽은 공간·표 위계 정리(웹 biz 화면)
