# 작업온 UI 디자인 방향 v1 (2026-07-11)

> 작성: Fable (디자인 디렉션). 시안 제작·검수는 별도 에이전트 수행.
> 근거: frontend-design 스킬 + ui-ux-pro-max 디자인 시스템 생성 결과를 현장 앱 특성에 맞게 조정.

## 1. 사용자·사용 환경 (디자인의 출발점)

- 주 사용자: 40~60대 장비기사·일용직·기술자. 스마트폰 익숙도 중간 이하도 포함.
- 사용 환경: **야외 강한 햇빛**(고대비 필수), 장갑 끼거나 거친 손(큰 터치 타겟), 이동 중 한 손 조작.
- 감정 맥락: 돈 문제(미수금)를 다루는 앱 — **신뢰감**이 최우선. 장난스럽거나 가벼워 보이면 안 됨.

## 2. 디자인 원칙

1. **햇빛 아래에서도 읽힌다**: 본문 17px 이상, 대비 4.5:1 이상, 회색-on-회색 금지
2. **장갑 낀 손으로도 눌린다**: 주요 버튼 높이 56px+, 터치 타겟 최소 48px, 간격 8px+
3. **한 화면 = 한 가지 일**: 화면당 주요 CTA 1개
4. **숫자가 주인공**: 금액·날짜는 크게, tabular numerals, 받을 돈(빨강 계열)/받은 돈(초록 계열)은 색+아이콘+텍스트로 구분 (색맹 대응)
5. 이모지 아이콘 금지 — Lucide 계열 SVG, stroke 2px 통일

## 3. 토큰

**컬러** (현장의 재료에서: 안전조끼 오렌지 + 작업복 네이비 + 서류 종이)
| 역할 | 값 | 의미 |
|---|---|---|
| Primary (브랜드/CTA) | `#F4770C` (안전 오렌지) | "온(ON)" — 켜짐, 현장의 색 |
| Ink (본문/헤딩) | `#1A2233` (딥 네이비) | 신뢰, 작업복 |
| Background | `#F7F6F3` (서류 종이톤) | 종이 확인서 메타포 |
| Surface | `#FFFFFF` | 카드 |
| 미수(받을 돈) | `#C2410C` | 텍스트+아이콘 병행 |
| 입금(받은 돈) | `#15803D` | 텍스트+아이콘 병행 |
| Border | `#E2DFD8` | 옅은 종이 경계 |
| 다크모드 | 네이비 기반(#0F1522 배경) + 오렌지 유지 | 새벽 출근 시간대 사용 고려 |

**타이포**: Pretendard 단일 패밀리 (한글 최적, Dynamic Type 대응)
- Display(금액/합계): 800, 28~34px, tabular-nums
- Heading: 700, 20px / Body: 400, 17px / Label: 500, 14px
- 과장된 초대형 타이포는 랜딩페이지에만, 앱 내부는 절제

**형태**: radius 14px(카드)/12px(버튼), 그림자 1단계만(0 1px 3px), 스페이싱 4/8 스케일

## 4. 시그니처 요소

**"종이 확인서" 카드**: 작업확인서를 실제 현장 먹지 확인서처럼 표현 —
상단 절취선(perforation) 디테일, 서명/도장 영역, 금액 큰 숫자.
이 카드 하나로 "아, 그 종이 확인서가 폰에 들어왔구나"를 즉시 이해시킨다.
나머지 UI는 조용하고 절제되게 유지 (시그니처에만 개성 집중).

## 5. MVP 화면 구성 (시안 대상 4종)

1. **홈**: 인사+오늘 날짜 → 오늘 일정 카드 → 이번 달 요약(일한 날/받을 돈/받은 돈) → 만료 임박 서류 경고 → 하단 탭 5개(홈/캘린더/+작성/장부/서류)
2. **확인서 작성**: 30초 목표. 이전 확인서 복사 버튼 최상단, 큰 입력 필드, 장비 섹션(토글), 단가→금액 자동 계산 표시, 저장&보내기
3. **장부**: 회사별 미수금 합계 리스트(금액 우측 정렬 tabular), 수금 예정 D-day 배지, 월 합계 헤더, 입금 체크 스와이프/버튼
4. **서류 지갑**: 서류 카드 그리드(썸네일+만료 D-day 배지), "묶음 보내기" 주요 CTA, 만료 임박은 상단 고정 경고

각 화면 목업은 375px 모바일 프레임, 라이트/다크 모두.
