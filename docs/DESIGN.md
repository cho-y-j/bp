## 1. 제품 정의

**"일한 것을 30초에 기록하면, 확인서·장부·정산·안전증빙이 자동으로 따라오는 앱"**

- 작업자(개인사업자·일용직·기술자·장비기사)에게: 작업확인서 + 받을 돈 장부 + 서류 지갑 + 만료 알림
- 소사업장(지시자)에게: 작업 확인·정산 + 안전관리 이행 증빙 리포트
- 핵심 차별점: **상대가 미가입이어도 링크 하나로 확인·서명 가능** (콜드 스타트 해결 + 성장 엔진)

## 2. 역할 모델 (단순하게)

```
사용자(user) 1명 = 프로필 1개
  ├─ 작업자 모드 (기본): 누구나 일한 것을 기록·청구
  └─ 사업장 모드 (선택): 사업장을 만들면 지시자 기능 활성화
```

- SKEP의 7개 역할 RBAC를 배제. **역할은 딱 2개(작업자/사업장)**, 한 계정이 둘 다 가능
  (예: 굴삭기 기사가 인부를 쓰면 그 순간 사업장 모드 사용 — 실제 현장에서 흔함).
- 사업장은 상호명 + 사업자번호(선택) + **초대코드(6자리)** 를 가짐.

## 3. 업종 포괄 설계 (핵심 구조 결정)

작업확인서를 "공통 코어 + 선택 섹션" 구조로 설계한다. 업종별 앱을 따로 만들지 않는다.

**공통 코어 (모든 업종):**
날짜, 현장/장소, 지시자(회사명), 작업자명, 작업 내용, 시작~종료 시각, 금액(단가 유형: 일당/시급/건당 + 연장·야간 등 추가), 특이사항, 서명

**선택 섹션 (작업자 프로필 설정에 따라 자동 표시):**
- 장비 섹션: 장비명, 차량번호, 규격, 유도원 여부 — **장비 기사가 켜면 확인서에 자동 포함**
- (확장 여지: 화물 섹션 — 상차지/하차지, 인력 섹션 — 인원수 등. v1은 장비 섹션만 구현)

**서류 지갑도 동일 원리:**
- 공통: 신분증, 사업자등록증, 통장사본, 자격증, 경력서 등 자유 업로드
- 장비 사용자 강화: 장비별 서류 세트(등록증·검사증·보험·면허) + **만료일 관리 및 D-30/D-7/당일 알림** (SKEP `has_expiry` 패턴 차용)

## 4. 기술 스택 (제안)

> **2026-07-11 확정**: 사용자 운영 중인 **자체 리눅스 서버에서 직접 호스팅** (Supabase 사용 안 함).
> 플랫폼: 안드로이드 + iOS + 웹 모두 출시. 로그인: 카카오 + 전화번호 인증. 스택 승인 완료.

| 레이어 | 선택 | 이유 |
|---|---|---|
| 모바일 앱 | **Flutter** (iOS+Android) | 한 코드로 양대 플랫폼, ref/skep과 같은 스택이라 양식·감 재활용 용이 |
| 웹 (링크 뷰어 + 사업장 웹) | **Next.js** | 미가입자가 설치 없이 확인서 열람·서명, 카톡 공유 미리보기(OG 태그) 필수라 웹 필요 |
| 백엔드 API | **NestJS (Node.js + TypeScript)** 모놀리식 단일 서비스 | 웹(Next.js)과 같은 언어로 유지보수 단순화. MSA 금지 원칙 (SKEP 교훈) |
| DB | **PostgreSQL 16** (Docker) | 단일 DB. SKEP 스키마 패턴 차용 용이 |
| 파일 저장 | 서버 디스크 (업로드 API 경유, Nginx 서빙) | v1은 로컬 디스크로 충분. 용량 증가 시 MinIO 전환 여지 |
| 배포 | **Docker Compose** (api + postgres + web + nginx) + Let's Encrypt | 사용자 리눅스 서버에서 직접 운영. 백업: pg_dump 크론 + 파일 rsync |
| 인증 | 카카오 OAuth + 전화번호 SMS 인증 (JWT 발급) | SMS는 쿨SMS/알리고 등 (건당 ~9원, 가입 시 1회라 부담 적음) |
| 푸시 알림 | **FCM** (무료) | 수금일·만료·예약·폭염 알림의 기본 채널 |
| 카카오 알림톡 | 2단계 도입 (건당 비용 발생) | 미가입 상대 전달·수신확인용. v1은 카톡 "공유하기"(무료)로 대체 |
| PDF 생성 | 백엔드에서 생성 (Puppeteer 또는 pdf-lib) | 확인서·명세서·안전리포트 일관 생성, 워터마크·마스킹 처리 |
| 이미지→PDF 변환 | 앱 내 처리(기울기 보정) + 서버 병합 | 어떤 이미지 포맷이든 업로드 → PDF 정규화 |
| 날씨 | 기상청 공공데이터 API (무료) | 현장 좌표 기준 폭염특보·체감온도 → 자동 안전 알림 |

## 5. 데이터 모델 (단일 PostgreSQL, 주요 테이블)

```
profiles          사용자: 이름, 전화(인증), 전화검색 허용 동의, 업종 태그
businesses        사업장: 상호, 사업자번호(선택), 초대코드, 소유자
connections       작업자↔사업장 연결: 상태(요청/수락), 연결 경로(전화검색/코드/QR/링크)
equipments        (장비 사용자) 장비: 종류, 차량번호, 규격
documents         서류: 소유자(프로필|장비), 파일, 유형, 발급일, 만료일, 상태
                  → expiry_date 인덱스, 만료 임박 조회 (SKEP 패턴)
jobs              작업 예약/지시: 사업장(또는 수기 상호명), 현장, 일시, 단가 정보, 상태
work_logs         수행 기록: 시작/완료 시각, GPS 스냅샷, 사진, 컨디션 체크
confirmations     작업확인서: 코어 필드 + 장비 섹션(JSONB), 금액 계산 내역,
                  share_token(외부 링크), 서명 이미지, 서명자 정보, 상태
                  상태: 작성됨 → 전송됨 → 서명됨 (SKEP 3단계 패턴 차용)
ledger_entries    장부: 확인서 연결, 받을 금액, 수금 예정일, 입금 기록(부분입금 허용), 상태
safety_logs       안전 증빙: 유형(폭염알림/휴식안내/서류유효확인/컨디션체크),
                  발송·수신·확인 시각, 대상자 — 수정 불가(append-only)
notifications     알림: 수금 D-day, 만료 D-30/7/0, 예약 확인, 폭염
```

**미가입 상대 처리 (중요):** `jobs`·`confirmations`의 상대방은 가입 사업장 FK **또는** 수기 입력(상호명+연락처)을 모두 허용. 나중에 그 사업장이 가입하면 수기 기록을 연결로 승격(전화번호 매칭). — 작업자 혼자서도 앱이 100% 동작하는 근거 구조.

**외부 서명 흐름:** 확인서 전송 → `share_token` 링크 생성(유효기간) → 카톡/문자 공유 → 상대가 웹에서 열람 → 이름 입력 + 터치 서명 → 서명 이미지 저장, 양측에 서명 완료 PDF. 링크 하단에 가입 유도.

## 6. 화면 흐름 (MVP)

**작업자 앱 (5탭):**
1. **홈** — 오늘 일정, 이번 달 수입/미수금 요약, 만료 임박 서류, 확인 대기 알림
2. **캘린더** — 일별 작업 기록·예약, 날짜 탭 → 확인서 작성
3. **확인서 작성 (+버튼, 30초 목표)** — 이전 확인서 복사, 자주 쓰는 현장/단가 저장, 장비 섹션 자동 포함 → 저장 즉시 장부 반영 → 전송(연결 사업장 or 링크 공유)
4. **장부** — 회사별 미수금 합계, 수금 예정일, 입금 체크(부분입금), 월간 명세서 PDF
5. **서류 지갑** — 업로드(자동 PDF화), 묶음 관리, 링크/카톡 전송, 만료일 등록·알림

**사업장 (웹 우선, 앱 내 모드 전환 동일 기능):**
- 수신함(확인서 확인·서명), 작업자 검색(전화번호)·초대, 작업 지시/예약, 작업 시작·완료 모니터(사진 포함), 정산 관리(미지급 집계·지급 체크), **안전관리 이행 리포트 PDF**

**외부(미가입) 웹 뷰:** 확인서 열람 + 서명 / 서류 묶음 열람 — 설치 불필요

## 7. 개발 단계

| 단계 | 범위 | 비고 |
|---|---|---|
| **Phase 1 — 작업자 단독 가치** | 확인서 작성·링크 서명, 장부, 서류 지갑(PDF 변환·공유·만료 알림), 캘린더, FCM 알림 | 상대 가입 불필요, 이것만으로 완결 |
| **Phase 2 — 사업장 연동** | 사업장 모드, 전화/코드 검색·연결, 작업 지시, 시작·완료 확인(사진), 정산 관리, 폭염 안전 알림 + safety_logs + 안전 리포트 | 성장 루프 완성 |
| **Phase 3 — 고도화** | 카카오 알림톡, 세금계산서 발행 API(바로빌/팝빌), 현장 QR, 월 자동 청구 | 비용 발생 기능은 여기로 |

## 8. 비기능 원칙

- **개인정보**: 전화번호 검색은 명시 동의자만 노출(법적 필수). 서류 링크는 유효기간+열람로그, 주민번호 마스킹, 워터마크 옵션
- **증거력**: safety_logs·서명 기록은 append-only(수정 불가), 시각은 서버 기준
- **비용**: 무료 티어 우선(Supabase/FCM/기상청 API), 건당 과금 기능(알림톡·세금계산서)은 Phase 3
- **오버빌드 금지**: 화면에 연결되지 않는 API는 만들지 않는다 (SKEP 교훈)
