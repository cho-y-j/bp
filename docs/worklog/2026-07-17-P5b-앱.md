### 1. 오늘의 출역 현황판 (`features/biz/attendance_board_screen.dart`)
- **사업장 홈 최상단 카드** `AttendanceBoardCard` — `GET /biz/today-attendance` 오늘(KST) 인원 요약 4칸(전체 ink / 출근 orange / 완료 green / 미출근 red·0이면 회색). 데이터 로드 전·실패 시 조용히 숨김(홈 방해 없음), 탭 → 상세.
- 상세 `AttendanceBoardScreen` — 전체 요약 + **현장별 그룹**(현장명·N명·완료 M) + 작업자 행: 마스킹 이름 · 상태 배지(예정/수락/시작/완료/취소 — 색 구분) · 시간 라인(시작했으면 "HH:mm 시작", 아니면 "HH:mm 예정") · **컨디션 배지**(OK=초록 웃는 얼굴 / BAD=warn 색, 색+아이콘+텍스트 병행). **당겨서 새로고침**(RefreshIndicator + invalidate).
- 요약 매핑 헬퍼 `attendanceSummaryCounts` — [전체,출근,완료,미출근] 순서 고정(테스트에서 화면 매핑 회귀 방지).

### 2. 현장별 인건비 (`features/biz/site_costs_screen.dart`)
- 사업장 메뉴 "현장별 인건비" — **기간 프리셋 칩**(이번 달 기본·최근 3/6/12개월, 백엔드 최대 12개월 제약 내) + `YYYY-MM ~ YYYY-MM` 라벨.
- `GET /biz/site-costs?from=&to=` → **전체 총계 헤더**(오렌지 틴트 카드: 총액 30px tabular·연인원·공수·확인서 건수) → **현장 카드**(소계·작업자 수·연인원, **탭 펼침**) → 작업자별 내역 행(마스킹 이름·**팀 배지 + 팀원 N명**·연인원·공수·금액 tabular). 연인원 소수(공수 1.5 등) 표기 위해 `siteCostsManDays` 라벨 신설(정수 라벨 `daysCount` 미사용).
- **PDF 저장·공유** — `GET /biz/site-costs/pdf` 인증 blob(`api.getBytes`) → 임시 파일 → `share_plus` 시스템 공유 시트(income_report_pdf 패턴 동일). 당겨서 새로고침·ErrorRetry·빈 기간 PaperCard 안내.

### 3. 지급명세서(월 마감) (`features/biz/wage_statement_screen.dart`)
- 사업장 메뉴 "지급명세서(월 마감)" — **월 선택**(‹ › 스텝) → `GET /biz/wage-statement?month=`.
- **소득유형 토글**(세그먼트: 사업소득 3.3% ↔ 일용근로) — 서버가 두 유형 세액을 동시 제공하므로 클라이언트 토글은 표시 전환만(재계산 없음). 총계 헤더(지급총액·지급 건수 + 선택 유형의 **원천징수 합계·차인지급액**, marked 면 **마감됨 배지**) → 작업자 카드(마스킹 이름·지급액·연인원·지급 건수 + 소득세/지방소득세/원천징수 합계(빨강)/차인지급액(초록) — 전부 tabular).
- **안내 노트 필수 표시** — 서버 `notes`(2026 기준 참고값·세무 상담 아님·**세무 전문가 확인 권장**) + `hometaxNote`(**주민번호 비수집·홈택스 직접 입력**) 를 warn 카드로 항상 렌더(빈 달에도).
- **복사** — 서버 `copyText`(홈택스 입력용) 클립보드 복사. **월 마감** — `POST /biz/wage-statement/mark`(멱등) → 성공/이미 마감 스낵바 분기, invalidate 후 marked 배지·버튼 비활성("마감됨"). 마감 후에도 조회 유지(백엔드 정책).
- 세액 요약 순수 함수 `wageWithholdingSummary(l, tax, lang)` — 테스트에서 표시 포맷 검증.
