# S3 웹 (Next.js 15) — 외부 공개 페이지 + 사업장 웹 (2026-07-11)

## 구현 범위
기존 web/ 스캐폴드(Next.js 15 App Router, 포트 3001) 위에 S3 전 범위를 구현했다.
백엔드(S2, 포트 3010 임시 pg 5433)와 실연동해 playwright 실브라우저 검증까지 마쳤다.

### 공통 인프라
- `lib/api.ts`: `NEXT_PUBLIC_API_URL` env. 두 경로 제공 —
  - `apiGet<T>()`: 서버 컴포넌트/SSR용 순수 fetch(`{data,error}` 봉투 언래핑, `no-store`).
  - `api()`: 사업장 웹 클라이언트용 **axios 인스턴스** — 요청 인터셉터(localStorage JWT →
    `Authorization`), 응답 인터셉터(`{data}` 언래핑, **401 → `/login?next=` 리다이렉트**).
  - `authedBlob()`: 인증 PDF를 blob 으로(안전 리포트). `absoluteUrl()`: 서버가 준 상대경로(pdfUrl/fileUrl)를 API 오리진 절대 URL 로.
- `app/globals.css`: **DESIGN-UI.md 토큰을 CSS 변수로** 정의(안전 오렌지 `#F4770C`, CTA 잉크
  `#241304`, 딥 네이비 `#1A2233`, 종이톤 배경 `#F7F6F3`, 미수 `#C2410C`/입금 `#15803D`,
  액센트 텍스트 `#B54708`). **다크모드는 `prefers-color-scheme`** 로 mockup-v1.html 과 동일한
  네이비 기반 값 미러. 본문 17px, CTA 56px, 터치 타겟 48px+, radius 12/14, 그림자 1단계.
- `components/`: `PaperConfirmation`(종이 확인서 시그니처 — 절취선·스탬프·필드표·큰 금액, 외부/수신함 공용),
  `SignaturePad`(**캔버스 터치 서명 직접 구현** — Pointer Events, 고DPI, PNG data URI, 라이브러리 0),
  `Icons`(Lucide 계열 SVG stroke 2px, 이모지 금지), `MonthNav`.

### A. 외부 공개 페이지 (모바일 우선, SSR)
- `/c/[token]` — 서버 컴포넌트가 `GET /public/confirmations/:token` fetch →
  종이 확인서 렌더 + **OG 메타**(`generateMetadata`, "작업확인서 — {현장명}", 금액·서명상태).
  미서명이면 서명 섹션(서명자명 + 캔버스) → `POST .../sign` → **완료 화면 + "작업온으로 관리하세요"
  가입 유도 배너 + 서명 PDF 링크**. 이미 서명됨/무효 토큰이면 각 상태 화면.
- `/s/[token]` — `GET /public/shares/:token` → 서류 카드 목록(유형·만료 D-day 배지·마스킹 표시)
  + 보기/다운로드(`.../files/:documentId`). 만료·무효 시 안내 화면.

### B. 사업장 웹 (데스크톱 겸용, 로그인 필요)
- `/login` — 전화번호 인증(request→verify), **dev 는 devCode 배너 표시**, JWT localStorage 저장.
- `app/biz/layout.tsx` — 사이드바(수신함/정산/작업자·지시/안전 리포트) + 사업장명 표시.
  토큰 없으면 로그인 리다이렉트, **사업장 미보유 시 생성 유도 화면**(BusinessGate). 모바일은 상단 탭바.
- `/biz/inbox` — `GET /biz/inbox` 목록(상태 배지) → 상세 모달(PaperConfirmation 재사용) →
  **같은 SignaturePad 로 앱내 서명** `POST /biz/confirmations/:id/sign`.
- `/biz/settlements` — 월 선택, 작업자별 미지급 집계, 체크 선택 → `POST /biz/settlements/pay`.
- `/biz/workers` — 전화번호 검색 → 연결 요청, 연결 목록, **작업 지시 폼**(`POST /jobs`, datetime→ISO).
- `/biz/safety` — 월 선택 + **안전 리포트 PDF(인증 blob → window.open)** + 최근 안전 알림 목록(notifications 필터).
- `/` 루트 — **랜딩으로 교체**(S1 검수 지적: `/`→API `/health` 리다이렉트 해소). `/health` 정적 페이지는 유지.

## 검증 (실제 실행 — 임시 pg 5433 + 백엔드 3010 + web dev 3001)

- **playwright 실브라우저**(모바일 375px + 데스크톱 1280px, 스크린샷 눈검수):

    **캔버스 서명(1902px 페인트 확인)** → 제출 → **SIGNED 화면 + 가입 배너**. 서버 status=SIGNED/signer=이현수 확인.
  - `/s/{token}`: 서류 2건(자격증 D-21 EXPIRING, 보험증권 D-173) 카드 + 파일 200 application/pdf. 무효 토큰 → 안내 화면.
  - 로그인(devCode 938652) → inbox → 상세 모달 → **앱내 서명(1784px) → 서명완료** → 정산에서 미지급 250,000 →
    선택 → **pay → 미지급 0/완납**. **작업자 장부 by-company 대칭 확인: 대성건설 paid 250,000 status=PAID**.
  - workers 검색(김*사, 굴삭기·토목) → **작업 지시 생성**(래미안…, 2026-07-15 08:00 KST → jobs SCHEDULED 확인).
  - safety: **인증 blob PDF 200/application/pdf/13,579B → window.open 새 탭**. 폭염 경고 알림 목록 표시.
  - **401/무토큰 → `/login?next=%2Fbiz%2Fsettlements` 리다이렉트**. **사업장 미보유(작업자 로그인) → 생성 유도 화면**.

## S3 독립 검수 지적 수정 (2026-07-11, 별도 검수 후 재작업)

구현과 분리된 독립 검수에서 나온 지적(중요 4·사소 4)을 수정하고 실기동 재검증했다.

### 중요
1. **SignaturePad 리사이즈 서명 소실** (`web/components/SignaturePad.tsx`) — 기존엔 `resize` 시
   서명을 초기화했다. 획을 **벡터 좌표(CSS px)로 보관(strokes)**하고, `resize`/`orientationchange`
   에서 캔버스 재초기화 후 **리드로우**하도록 변경. 모바일 주소창 접힘/키보드/회전에도 보존.
2. **공개 SSR 장애 처리** (`web/lib/api.ts`, `app/c`·`app/s`) — `apiGet` 에 **AbortController 8초
   타임아웃** + 네트워크/타임아웃을 status=0 ApiError 로 구분. `classifyLoadError()` 로
   **4xx=무효/만료 토큰 → `notFound()`(HTTP 404)**, **네트워크·타임아웃·5xx=일시장애 → 친화 화면(HTTP 200)**.
   `app/error.tsx`·`app/global-error.tsx`·`app/not-found.tsx`(`components/StatusScreen.tsx`) 추가.
3. **수신함 모달 상세** — 백엔드 `GET /biz/confirmations/:id` 추가(소유자+자기 사업장 대상만,
   403/404 강제. `ConfirmationsService.bizConfirmationDetail`). 목록은 요약만 담으므로 **모달 오픈 시
   상세 조회** → 시간(`08:00~17:30`)·금액 내역(`기본(일당) 250,000×1`) 완전 렌더(이전엔 `-~-`, 단일 합계행).
4. **/s 다운로드 실동작** — 백엔드 public 파일 스트림에 **`?download=1` → `Content-Disposition: attachment`**
   지원(기본 `inline`). 웹 다운로드 버튼이 `?download=1` URL 사용.
