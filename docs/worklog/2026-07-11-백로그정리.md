## 구현 요약

### 1. 공유 링크 무효화(revoke) 엔드포인트 — 확인서·계약서 일관 공백 해소
기존엔 `revokedAt` 공개열람 차단 가드는 있으나 트리거 엔드포인트가 `document-shares`(DELETE)에만 있었다. 확인서·계약서에 발행자/사업장용 revoke 를 추가.

- **`POST /confirmations/:id/revoke`**(발행자=확인서 소유 작업자) — `ConfirmationsService.revoke`.
- **`POST /biz/contracts/:id/revoke`**(사업장 소유자) — `LaborContractsService.revoke`.
- **정책(양쪽 동일)**: **SENT 만 무효화 가능** → `revokedAt` 설정 후 public 열람/서명 403.
  **SIGNED 는 무효화 불가 → 409**(서명 완료 문서는 증빙 보존을 위해 **링크 열람 유지**). **DRAFT(미전송) → 409 NOT_REVOCABLE**. 재호출은 멱등(이미 revokedAt 이면 재설정 안 함).
  에러코드: 확인서 `CONFIRMATION_REVOKED`(public 403)/`ALREADY_SIGNED`/`NOT_REVOCABLE`, 계약서 `LABOR_CONTRACT_REVOKED`/`ALREADY_SIGNED`/`NOT_REVOCABLE`.
- 권한 격리: 확인서는 `ownedOrThrow`(profileId), 계약서는 `ownedContractOrThrow`(business.ownerId) → 남은 404.
- `document-shares` DELETE 무효화는 이미 존재(확인만, 무변경).
- **웹/앱 UI 노출은 백로그 유지** — API 만 제공(정책 확정).

### 2. 웹 다중 사업장 전환 UI (S3 백로그)
- `web/app/biz/layout.tsx`: `businesses/mine` 전체 로드 → **소유 2개 이상이면 사이드바 상단(+모바일 topbar)에 사업장 선택 드롭다운**(`BusinessSwitcher`, 네이티브 `<select>` 접근성). 선택값 `localStorage['jakeobon_selected_biz']` 저장·복원(목록에 없으면 첫 사업장). **단일 사업장이면 드롭다운 미노출**(기존 라벨 유지). 로그아웃 시 선택값 제거.
- `web/app/biz/biz-context.tsx`: 컨텍스트에 `businesses`·`selectBusiness` 추가, `business` = 선택된 사업장.
- inbox/settlements/safety/contracts 페이지: `useBiz().business.id` 를 쿼리 파라미터로 전달, `business.id` 변경 시 재조회.
- **백엔드는 additive 쿼리 파라미터로 스코프 지원**(기존 응답은 소유 전체 집계였음):
  `GET /biz/inbox?businessId=`·`/biz/settlements?month=&businessId=`·`/biz/safety-report?month=&businessId=` — `BizService.scopedBusinessIds(userId, businessId?)`(미지정=소유 전체, 지정&소유=해당 1건, 지정&미소유=빈 배열→유출 차단). `pay` 는 스코프 무관(소유 전체 ledger 대상).
- `web/app/biz/workers/page.tsx`: 이미 `useBiz().business.id` 로 연결·작업지시 대상 사업장을 쓰므로 전환에 자동 반응(무변경).
