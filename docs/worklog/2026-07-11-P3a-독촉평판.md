# P3a — 수금 독촉 자동화 + 지급 평판 배지 (2026-07-11)

> STRATEGY.md P3-신규 1·2 구현. 작업자(사용자)·사업주 양쪽 편리성 최우선, 부정 낙인 금지(좋은 것만 노출).
> 포트: 임시 pg 5436 · 백엔드 3040 · web 3004 만 사용. live-db(5432) 무손상.

## 1. 무엇을 / 왜

### ① 수금 독촉 자동화
작업자가 버튼 하나(자동 안내 ON)만 켜면, 앱이 작업자를 대신해 상대(사업장/수기 미가입 상대)에게 수금일 이후 D+7/D+30에 점잖은 대금 안내를 자동 발송. 수동 즉시 발송 버튼도 제공(쿨다운 3일). 체불 페인포인트 완화 + 독촉의 감정 부담 제거.

### ② 지급 평판 배지
정산 데이터(SIGNED→전액 PAID 소요일)로 사업장 "평균 지급 소요일"을 자동 산출 → 우수(≤15일)/양호(≤30일) 배지. 주관 리뷰 없는 데이터 기반. 검색·프로필에 노출, 사업장 본인은 개선 안내 확인. 부정 낙인 금지(>30일·데이터 부족은 배지 없음, 안내만).

## 2. 백엔드 (구현·검증 완료)

### 스키마 (마이그레이션 1개: `20260711100000_p3a_reminders_badge`, additive)
- `LedgerEntry`: `autoRemind Boolean @default(false)`, `reminders Json[] @default([])`(발송 이력 [{at, channel, stage}], stage=D7|D30|MANUAL). Business 연결·수기 상대 모두 지원.
- `Business`: `paymentAvgDays Float?`, `paymentSampleSize Int @default(0)`, `paymentBadgeUpdatedAt DateTime?` (배지 캐시 — 검색 실시간 집계 금지).
- `Profile`: `payoutBank/payoutAccount/payoutHolder String?` (수금 안내용 계좌, 선택 입력).

### 수금 독촉
- `PATCH /ledger/:id` 에 `autoRemind` 토글 additive(파생 항목은 기존 409 LEDGER_DERIVED_READONLY로 차단).
- `POST /ledger/:id/remind` — 수동 즉시 발송. 쿨다운 3일 초과 시 발송, 이내면 409 REMIND_COOLDOWN(남은 시간 안내). 완납 409 LEDGER_ALREADY_PAID, 대상 없음 409 REMIND_NO_TARGET, 타인 404, 파생 409.
- 크론(`LedgerDueScheduler` 확장, 매일 10:00 KST): `autoRemind=true`·미수·비파생·수금예정 D+7/D+30 도달 항목 → 발송. **연결 사업장 = 사업장 소유자에게 푸시+인앱 알림**(NotificationType.PAYMENT_DUE + data.kind='reminder'), **수기 상대 = 알림톡**(전화 있을 때, `PAYMENT_REMINDER` 템플릿 신설, 확인서 공개 링크 `/c/{shareToken}`+금액+계좌 포함). 발송 이력 append, **같은 단계 중복 발송 방지**.
- 문구(점잖게): "○○님이 ○월 작업 대금 안내를 드립니다. 금액 …원 (은행 계좌 예금주)". 계좌는 작업자 프로필 payout 필드.
- 알림톡 어댑터: `AlimtalkTemplateKey` 에 `PAYMENT_REMINDER` 추가(키 없으면 로그만, 기존 안전 동작 유지). env `ALIMTALK_TEMPLATE_PAYMENT_REMINDER` 추가.

### 지급 평판 배지
- 집계(`BadgeService`): 사업장별 SIGNED 확인서 중 연결 장부가 전액 PAID 인 건의 (signedAt→전액지급 시각) 일수 평균, 최근 12개월, **표본 3건 이상일 때만** 산출. 등급 우수(≤15)/양호(≤30)/표시없음. 전액지급 시각 = payments 시간순 누적이 처음 amount 이상이 된 결제 paidAt.
- 캐시: `Business` 컬럼에 저장. **일일 크론**(04:00 KST) 전 사업장 재계산 + **pay 시점 비동기 갱신**(`recomputeBusinessQuietly`, fire-and-forget).
- 노출: `GET /businesses/search`·`GET /businesses/:id`(신설) 응답에 `paymentBadge {grade,avgDays,sampleSize}|null` additive(우수/양호만). 사업장 본인 `GET /biz/payment-badge` — status(EXCELLENT|GOOD|NONE|INSUFFICIENT)+개선 안내.
- 순수 유틸(`badge.util.ts`): gradeForAvgDays 경계·computeAvgDays 표본·badgeFromCache(우수/양호만)·selfBadgeStatus. `reminder.util.ts`: autoReminderStage(D7/D30)·hasStageBeenSent·manualCooldown.

## 5. P3b(QR 명함)에 넘길 주의사항
- 공개 링크 규약: 확인서 공개 페이지 `/c/{shareToken}` 를 독촉 알림톡에 재사용 중. QR 명함의 공개 프로필 페이지는 별도 경로(예 `/p/{token}`)로 두어 충돌 회피.
- Profile 에 payout 계좌 필드 신설됨 — QR 공개 프로필에 계좌 노출은 신중히(민감정보, 기본 비노출 권장).
- `paymentBadge`(우수/양호만) DTO 는 QR 공개 프로필의 "지급 평판" 뱃지로 재사용 가능(이미 캐시·크론 구축됨). 단 공개 페이지는 인증 없이 접근하므로 badgeFromCache 만 노출하고 내부 avgDays 원본 과다 노출 주의.
- 알림톡 `PAYMENT_REMINDER` 템플릿은 카카오 비즈메시지 승인 필요(env 미설정 시 로그만) — 운영 배포 전 승인 절차(DEPLOY.md) 대상 추가.
