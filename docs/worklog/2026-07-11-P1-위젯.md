# P1 — 홈 화면 위젯 (2026-07-11)

> 작업자·사업주 양면 편의성 관점의 P1 마지막 배치. `home_widget` 패키지 기반
> iOS(WidgetKit) + Android(AppWidget) 홈 화면 위젯. 위젯은 네트워크 호출 없이
> 앱이 공유 저장한 값을 렌더만 한다.

## 무엇을 / 왜
- **오늘 일정**(첫 작업 현장명·시간, 없으면 "오늘 일정 없음") + **이번 달 미수금**(금액, 미수 빨강)을
  앱을 열지 않고 홈 화면에서 즉시 확인 (STRATEGY.md P1 정의).
- 소형(2x2, 미수금 중심) / 중형(4x2, 일정+미수금) 두 사이즈. 탭 → 앱 열기.
- 문구는 **앱 언어 설정**을 따른다 → 로케일 문자열을 앱에서 렌더해 공유 저장, 네이티브는 렌더만.
- 로그아웃 시 위젯을 "로그인해 주세요" 상태로 클리어.

### 미검증 (환경 제약 — 정직 보고)
- **iOS 홈 화면에 위젯을 실제 배치한 스크린샷 / 다크모드 위젯 / 탭→앱 열기**: 미확보.
  이 환경은 시뮬레이터 SpringBoard UI 자동화가 불가하다 — osascript(System Events)에 **보조 접근(Accessibility)
  권한 없음**(-1719), `cliclick`/`idb` 미설치, `simctl` 에 위젯 배치 API 없음, 창 좌표 질의도 AX 필요.
  위젯을 홈에 추가하는 롱프레스→갤러리→검색→추가 플로우를 자동화할 수단이 없다.
  → **P1 통합 검수(GUI 보유)에서 사람이 위젯을 홈에 추가해 소형/중형·라이트/다크·탭 동작을 눈검수**할 것.
  공유 데이터 파이프라인과 확장 임베드/설치는 위와 같이 실측 완료.
- **Android 런타임**: 에뮬레이터 미설치(`emulator not found`) → 빌드 통과 + 레이아웃/Provider 코드 리뷰로 대체(태스크 허용).
