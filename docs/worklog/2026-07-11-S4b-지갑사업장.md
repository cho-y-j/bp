# S4b 서류 지갑·사업장 모드·알림 (Flutter, 앱 마지막 배치) — 2026-07-11

## 구현 범위
S4a(코어) 위에 서류 지갑·사업장 모드·알림을 얹어 작업온 앱을 완성했다. 기존 컨벤션
(ThemeExtension `context.c`, PaperCard, ApiClient 봉투 언래핑/JWT/`getBytes` blob, Riverpod
FutureProvider, share_plus) 을 그대로 재사용했다. 백엔드(임시 pg **5435** / API **3030**)와
실연동해 iOS 시뮬레이터(iPhone 17, iOS 26.5)에서 E2E 검증까지 마쳤다.

### 1. 서류 지갑 (`features/wallet/`)
- `wallet_screen.dart`: 서류 카드 그리드(유형 아이콘 + 만료 D-day 배지 4색, 마스킹 표시),
  상단 만료 임박 고정 경고(WarnBanner), 롱프레스 다중 선택 → **묶음 보내기(주요 CTA)**.
- `upload_sheet.dart`: 소스 선택(카메라/갤러리 `image_picker` · PDF `file_picker`) → 유형 선택
  (신분증/사업자등록증/면허·자격증/차량·검사증/보험/기타) + **장비 연결 옵션** + 만료일 입력 →
  multipart `POST /documents`. 업로드 직후 마스킹 편집 유도.
- `mask_editor.dart` + `core/mask_geometry.dart`: 문서 이미지 미리보기 위에 **드래그로 사각형**
  지정 → `normalizeDragRect()` 로 정규화 좌표(0~1, 역방향/이탈 clamp) → `POST /:id/mask`.
- `document_detail_screen.dart`: 인증 blob 미리보기(이미지 `Image.memory`, PDF `open_filex`),
  만료일 수정, 마스킹 편집 재진입, 삭제.
- 묶음 공유: 다중 선택 → 유효기간(7/14/30일) 선택 + 마스킹본 여부 안내 → `POST /document-shares`
  → **시스템 공유 시트**(share_plus). `my_shares_screen.dart`: 열람 횟수·만료·무효화(revoke).
- `equipment_screen.dart`: 장비 CRUD + 장비별 서류 추가.

### 2. 사업장 모드 (`features/biz/`, `features/jobs/`)
- `business_mode_screen.dart`: `me.hasBusiness` 분기 — 없으면 생성 플로우(상호/사업자번호),
  있으면 사업장 홈(수신함/정산/작업자·지시/작업 지시 목록/안전 + 초대코드).
- 수신함: `inbox_screen.dart` → `biz_confirmation_detail_screen.dart`(`GET /biz/confirmations/:id`,
  PaperCard 렌더) → **앱내 서명** — `widgets/signature_pad.dart`(캔버스 SignaturePad 를 Flutter 로
  이식: 획 벡터 보관·리사이즈 보존) + `core/signature.dart`(PNG 인코딩 → `{signerName,
  signImageBase64}`, ≤1MB 보정 재인코딩).
- `settlement_screen.dart`: 월별 작업자 미지급 집계 → 선택 지급(`POST /biz/settlements/pay`),
  지급 후 작업자 장부 대칭 반영(invalidate).
- `workers_screen.dart`: 전화 검색(`/workers/search`) → 연결 요청/수락 → **작업 지시 생성**
  (현장/일시 picker/단가유형·단가, `scheduledAt` ISO8601). `jobs_screen.dart`: 지시 목록.
- `safety_screen.dart`: 안전 리포트 PDF(인증 blob → open_filex).
- 작업자 측 `my_jobs_screen.dart`: 받은 작업 수락(confirm)·시작(start: `geolocator` GPS,
  **권한 거부/지연 시 (0,0) 로 진행** + 컨디션 체크 다이얼로그)·완료(complete).

### 3. 알림 (`features/notifications/`, `core/push.dart`)
- `notifications_screen.dart`: `GET /notifications` 목록·읽음 처리, 폭염 알림에 **"확인" 버튼**
  (`POST /safety/:logId/ack`). 홈 벨 아이콘 **미읽음 뱃지**(`unreadCountProvider` + `badgeCount()`).
- FCM: `firebase_core`/`firebase_messaging` 의존성 + `PushService`(토큰 등록 `POST /device-tokens`).
  **Firebase 설정 파일 없으면 `initializeApp()` 예외를 잡아 skip 하고 앱 정상 동작**(키 미보유).
- 더보기 설정: **전화검색 동의 토글**(`PATCH /me` phoneSearchConsent).

## iOS/시뮬레이터 이슈와 해결 (정직 보고)
- **Firebase pods**: `firebase_core`/`firebase_messaging` 추가로 iOS 배포타깃을 **15.0**(Podfile) 로
  상향. `flutter build ios --simulator` 로 pod 정상 설치·빌드(80s). 설정 파일 미보유라 런타임
  초기화는 skip(의도).
- **GPS 블로킹**: 시뮬레이터에서 `getCurrentPosition` 이 픽스 지연으로 무한 대기 → 시작 버튼이
  스피너로 멈춤. `tryGetPosition()` 을 **전체 6초 하드 타임아웃**으로 감싸 UI 를 절대 막지 않도록
  수정(운영에도 유효). 시뮬레이터 `simctl privacy grant location` + `location set` 로 검증.
- **서명 버튼 지연 빌드**: 상세를 `ListView`(지연 빌드)로 두면 하단 서명 버튼이 빌드 안 됨 →
  `SingleChildScrollView`+`Column` 으로 변경, 테스트는 `ensureVisible`(프로그램 스크롤, 제스처
  충돌 없음) 로 서명 영역을 올림.
- **스크린샷 blank**: 동일 시뮬레이터에서 연속 `flutter drive` 시 2회차부터 `takeScreenshot` 이
  흰 화면(surface 캡처 상태) → **부팅 직후 1회 실행**이면 정상. 최종 검증은 sim 재부팅 후 단일
  실행으로 14장 실캡처.
- 네이티브 피커/공유 시트는 WidgetTester 로 조작 불가 → 파일 선택은 provider override 로 주입,
  묶음 공유는 시트 옵션 화면까지 캡처 후 링크 생성(API 로 생성 확인).
