# 거래처(Partners) 2단계 — 수동 추가 + 전송 후 저장 제안 (backend + app)

일자: 2026-07-18
범위: backend + app (web 무수정). 1단계(커밋 cbc17df, `worklog/2026-07-18-거래처-1단계.md`)의 연장.

## 배경 (사용자 지적)
1단계 자동 수집은 **확인서 상대만** 잡는다. 확인서를 쓴 적 없는 거래처(문자로 서류만 보내는 곳,
추후 세금계산서 발행 대상)를 등록할 방법이 없었다 → **수동 추가 + 전송 시 저장 제안**으로 공백을 막는다.

## 구현

### 1) backend — `POST /partners`
- body: `{ name(필수), phone?, alias?, bizNumber?, email?, memo? }`. 본인 소유로 생성.
- **중복 처리 = 409** (upsert 아님). 근거(주석 명시):
  - 자동 수집 훅(`upsertFromManualCounterparty`)은 name/phone 만 만지는 idempotent 백그라운드
    동작이지만, 이 POST 는 사용자가 **명시적으로 "추가"** 하는 행위이고 alias/bizNumber/email/memo
    까지 실어 온다. 같은 이름의 기존 행을 조용히 덮으면 이미 채워둔 보강값을 파괴할 수 있다.
  - 이미 있는 이름을 다시 넣는 것은 실수일 가능성이 높아 409 로 알려 주는 편이 UX 상 낫다
    (앱은 "이미 등록된 거래처예요"로 처리). 자연 키 (profileId, name) 유일성은 두 경로에서 동일.
  - 구현: `prisma.partner.create` → P2002 catch → `AppException('PARTNER_DUPLICATE', 409)`.
- 유효성: name trim 비어있으면 400(`PARTNER_NAME_REQUIRED`). phone 은 자유 형식
  (수기 확인서 `contact` 규칙 재사용 — 별도 정규식 없음, `@MaxLength(50)`). 빈 문자열 → null 정규화.
- 응답: 확인서 0건이므로 통계 0 동형(GET 아이템과 같은 shape: confirmationCount 0·outstanding 0·
  paid 0·lastWorkedDate null). 앱이 곧바로 목록에 반영 가능.

### 2) app — 거래처 목록 "+ 추가"
- `PartnersScreen` 에 FAB `FloatingActionButton.extended`("거래처 추가") → `_AddPartnerSheet`
  (이름 필수 + 전화·사업자번호·이메일·메모 선택) → 저장 성공 시 `invalidate(partnersProvider)` + 스낵바.
  409 → `partnerDuplicate` 문구, 그 외 → 서버 메시지.
- **확인서 0건 카드 표기**: `noRecord = confirmationCount==0 && outstanding<=0` 이면 우측 금액/정산완료
  대신 **"기록 없음"**, 하단 행은 전화번호(있으면)만 표시(기존 카드 스타일·여백 유지).

### 3) app — 전송 후 저장 제안
- 수신인 번호 **출처 추적**(`_RecipientSource` enum): manual·contactPicker·partnerChip·teamChip·preset.
  - 전화 TextField `onChanged` → manual (칩/피커는 controller.text 프로그램적 세팅이라 onChanged 미발생).
  - 거래처 칩 → partnerChip, 팀원 칩 → teamChip, 연락처 피커 → contactPicker, 프리셋(있으면) → preset.
- `composeSms` 복귀 후 `_maybeSuggestSavePartner`:
  - manual/contactPicker 만 제안(거래처 칩·팀원 칩·프리셋=이미 아는 상대는 제외).
  - 번호 정규화(숫자만) 8자리 미만 skip / 기존 거래처 번호와 일치 시 skip.
  - **같은 번호 재제안 방지**: `PartnerPromptStore`(SharedPreferences seen-set, 띄우기 전에 기록 →
    닫아도 재노출 안 함, 최근 200개 유지).
  - 다이얼로그(`_SavePartnerDialog`): 이름 1필드(연락처 피커면 이름 프리필, 직접 입력이면 비어 있음 →
    비우고 저장 시 전화번호를 이름으로 대체). 저장 → create → invalidate + 스낵바. 저장 중 409 →
    조용히 닫음(이미 거래처이므로 목적 달성).
