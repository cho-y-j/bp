# P1 다국어 — 외부 공개 웹 (2026-07-11)

> 범위: **web/ 외부 공개 페이지만** (확인서 열람·서명 `/c`, 서류 묶음 `/s`, 404/오류 StatusScreen, 가입 유도 배너).
> 앱(app/)·백엔드(backend/)는 **미수정**(백엔드는 읽기만). 사업장 웹/로그인은 대상 제외.
> 포트 격리: 임시 pg **5437**, 백엔드 **3050**, web dev **3004**. live-db(5432)·5436·3040 미접속.

## 무엇을 / 왜
현장 외국인 노동자가 알림톡 링크로 받은 **작업확인서를 자기 언어로 읽고 서명**하도록, 외부 공개 페이지를 6개 언어로 다국어화.
지원 언어(리서치 확정, `worklog/2026-07-11-외국인노동자-조사.md`): **한국어(기본) + 중국어 간체·러시아어·베트남어·네팔어·영어**.

## 어떻게 (설계)
- **경량 사전(dictionary) 방식** — 무거운 i18n 라이브러리 없이 SSR·클라이언트 공용.
  - `web/lib/i18n/{ko,zh,ru,vi,ne,en}.ts` — `ko.ts` 가 기준(source of truth). `type MessageKey = keyof typeof ko` 로 **다른 5개 언어 파일이 모든 키를 채우도록 컴파일 타임 강제**(Record<MessageKey,string>).
  - `web/lib/i18n/index.ts` — `t(lang,key,params)`(‹{param}› 치환·미번역 키는 ko 폴백), `createT(lang)`, `Lang`, `LANGS`, `LANG_NATIVE`(자국어 표기), `LANG_LOCALE`(Intl), `resolveLang`(순수), `normalizeLang`, `pickFromAcceptLanguage`(q-weight 파싱).
  - `web/lib/i18n/server.ts` — `resolveServerLang(explicit?)`: Next15 async `cookies()`/`headers()` 로 SSR 언어 결정.
- **초기 언어 우선순위**: ①`?lang=`(알림톡 딥링크) → ②저장값(쿠키 `jakeobon_lang`) → ③`Accept-Language` → 기본 ko.
- **언어 전환 UI** — `web/components/LanguageSwitcher.tsx`(client): 지구본(신규 `Globe` 아이콘)+현재 언어(자국어), 드롭다운. 시안 디자인 토큰 스타일(`.lang-*` CSS, surface/border/accent 토큰·다크모드). 선택 시 **쿠키+localStorage 저장 + `?lang=` 갱신 + `router.refresh()`** 로 SSR 문구 전환. 마운트 시 `current` 를 저장하고 `document.documentElement.lang` 동기화(→ `?lang=` 딥링크 진입도 자동 기억·html lang 보정).
- **번역 대상**: UI 문자열 전부(제목·필드 라벨·버튼·안내·오류·서명 안내·**서명의 법적 의미 안내 신규 문구** `signLegal`·가입 유도). **사용자 입력 데이터(현장명·회사명·작업내용·차량번호·서명자명)는 미번역.** 금액 항목 라벨(기본/연장/조출/야간/철야)은 서버 `item.type` 기준으로 프런트에서 번역(OTHER·미지 유형은 서버 label 폴백), 한국어는 서버 라벨 그대로 유지.
- **포맷(Intl)**: `web/lib/format.ts` 에 `formatDate(input,lang)`(요일 포함 로케일 날짜)·`money(n,lang)`·`grouped(n,lang)` 추가. 통화는 **원화 고정** — 한국어 `1,234원`, 그 외 `₩1,234`(언어별 천단위: ru 공백, ne 인도식 라크 그룹핑+데바나가리 숫자 등 로케일 규칙 그대로).
- **폰트**: `globals.css` body 폰트 스택에 데바나가리 폴백 추가(`Noto Sans Devanagari`/`Kohinoor Devanagari`/`Devanagari Sangam MN`/`Nirmala UI`/`Mangal`). Pretendard 는 한글·라틴·**키릴·베트남어** 커버. `<html lang>` 은 layout 에서 쿠키/Accept-Language 로 동적, `?lang=` 딥링크는 스위처 마운트가 보정.
- **OG 태그는 한국어 유지**(발신자=한국 사업장/작업자, 수신자 카톡 미리보기) — `generateMetadata` 미변경.

### 변경/추가 파일 (web/ 만)
- 신규: `lib/i18n/{index,server,ko,zh,ru,vi,ne,en}.ts`, `components/LanguageSwitcher.tsx`
- 수정: `lib/format.ts`(+money/grouped/formatDate), `components/Icons.tsx`(+Globe), `components/PaperConfirmation.tsx`(lang prop·항목/필드/통화/날짜 번역), `components/SignaturePad.tsx`(hint/ariaLabel prop, ko 기본값→biz 무변경), `components/StatusScreen.tsx`(lang prop·스위처), `app/layout.tsx`(동적 html lang), `app/c/[token]/page.tsx`, `app/c/[token]/SignSection.tsx`, `app/s/[token]/page.tsx`, `app/not-found.tsx`, `app/error.tsx`, `app/globals.css`(+.lang-* 스타일·데바나가리 폴백)
- 공유 컴포넌트(PaperConfirmation/SignaturePad)는 `lang`/`hint` 기본값 ko → **사업장 웹(biz) 렌더는 한국어 그대로**(통화 표기만 `200,000 원`→`200,000원` 미세 변경).

## 검증 (실측)
임시 pg(5437, docker postgres:16) + 백엔드(3050, `node dist/main`, NODE_ENV=development) + web dev(3004). `prisma migrate deploy` 후 API 로 시드: 작업자 가입→확인서(일당 200,000+연장 60,000+VAT 10%=**286,000**, 장비+유도원)→send(=/c 토큰), 서류 업로드(PNG→PDF)→document-shares(=/s 토큰).

- **`npm run build` 타입에러 0** (dev 서버 정지 후 clean 빌드 재확인). 6개 라우트 dynamic 렌더.
- **playwright(모바일 390×844) 6개 언어 `/c` 전환 스크린샷 Read 눈검수**:
  - `i18n-c-ko.png`·`i18n-c-zh.png`·`i18n-c-ru.png`·`i18n-c-vi.png`·`i18n-c-ne.png`·`i18n-c-en.png` (repo 루트)
  - **네팔어(데바나가리)·러시아어(키릴) 글자 깨짐 없음** 확인. 네팔어는 로케일대로 데바나가리 숫자+라크 그룹핑(₩२,८६,०००), 러시아어는 공백 천단위(₩286 000), 베트남어 성조(dấu) 정확.
  - 데이터(강남 현장 A동·박민수·대한건설·굴착기)는 각 언어에서도 한국어 유지 = 번역 대상 분리 정상.
- **`?lang=vi` 직접 진입**: html lang→`vi` 보정, 쿠키 `jakeobon_lang=vi` 저장, 스위처 "Tiếng Việt".
- **언어 유지(새로고침)**: `?lang` 없는 `/c` 재진입 시 쿠키(vi)로 SSR 베트남어 렌더 확인.
- **스위처 UI**: 드롭다운 6개 언어 자국어 표기(네팔어 नेपाली 포함) 정상(`i18n-switcher-open.png`), 클릭→언어 전환 동작.
- **베트남어 상태에서 서명 제출 성공**: 이름 "Nguyễn Văn A"+캔버스 서명→제출→"Đã ký xong" 완료+가입 배너(`i18n-c-vi-signed.png`). 백엔드 `public/confirmations/:token` 재조회 `signed:true, signerName:"Nguyễn Văn A"` 대조 확인.
- **`/s` 2개 언어**: 영어(`i18n-s-en.png`)·네팔어(`i18n-s-ne.png`) — 제목·유효기간·만료 D-day·보기/다운로드 번역. 서류 유형("자격증")은 데이터로 미번역.
- **404 StatusScreen 2개 언어**: 네팔어(`i18n-404-ne.png`)·러시아어(`i18n-404-ru.png`). 404는 Next15 가 not-found 에 searchParams 를 주지 않아 `?lang=` 무시하고 **쿠키/Accept-Language 기반**(알림톡 링크는 항상 유효 토큰 → 실사용 영향 없음). 스위처(쿠키+refresh)로 전환됨 확인.
- console error 0(404 fetch 로그 제외).
- **자원 정리**: web dev·backend 종료, 임시 pg 컨테이너 삭제, `web/.env.local` 원복(3010). jakeobon 관련 컨테이너 0. **live-db(5432) 무접속·무손상**(전 과정 5437/3050/3004 만 사용).

## 번역 품질 — 원어민 검수 권장(미검수 항목, 정직 보고)
기계/모델 번역이며 원어민 검수 전. 특히 아래는 확인 권장:
- 법률 문구 `signLegal`(서명의 법적 효력 안내) — 각국 노동 관행상 표현 강도.
- 직역 용어: `paperGuide`(유도원) ne `निर्देशक`/ru `Сигнальщик`/vi `Người hướng dẫn`, `paperOrderer`(지시자) 각 언어 뉘앙스, 금액 라벨 `조출(EARLY)`.
- 네팔어 존칭/구어체 자연스러움, 베트남어 성조 최종 확인.
- 서류 유형(자격증 등)·현장/회사명은 **의도적으로 미번역**(데이터).

## 앱 다국어 배치에 넘길 것(재사용)
- **사전 구조 재사용**: `ko.ts` 기준 키 + `Record<MessageKey,string>` 완전성 강제 패턴을 Flutter(intl/ARB)로 이식. `MessageKey`·문구를 ARB 키로 1:1 매핑 가능(브랜드 `작업온`·통화 규칙 `₩`/`원`·Intl 로케일 매핑 표 `LANG_LOCALE` 공유).
- **언어 결정 우선순위**(?lang→저장값→시스템 로케일→ko)·초기 언어 로직 동일 적용.
- **데이터 vs 라벨 분리 규칙**(현장/회사/작업내용/서명자명 미번역, 금액 항목은 type 기준 번역) 그대로 준수.
- 확인서 PDF 이중언어 옵션(백엔드)은 별도 — 본 작업 범위 아님.
- 2차 확장 언어(크메르·인니·태국·미얀마·몽골)는 파일 추가만으로 확장 가능(스위처·해석기 무변경).
