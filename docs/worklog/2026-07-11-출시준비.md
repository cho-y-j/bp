## 1. 앱 아이콘 + 스플래시

- 스크립트: `app/tool/gen_brand_assets.py` (재현 가능). 산출물 `app/assets/branding/`:
  - `icon_master.png` (1024, 오렌지 #F4770C full-bleed + 흰 "온" — web/app/icon.svg 파비콘과 통일)
  - `icon_foreground.png` (1024, 투명 + 흰 "온", Android adaptive 안전영역)
  - `icon_dark.png` (1024, 딥 네이비 #1A2233 + 오렌지 "온" — iOS 다크)
  - `icon_tinted.png` (1024, 투명 + 흰 "온" — iOS 틴트/모노크롬, 루미넌스 기준)
  - `splash_icon.png` (1152, 오렌지 라운드 "온" — 종이톤 위에 얹힘)

### 생성 도구 적용
- `flutter_launcher_icons ^0.14.1` + `flutter_native_splash ^2.4.3` (dev_dependencies) 추가, 설정은 `pubspec.yaml` 내.
- `dart run flutter_launcher_icons` → iOS 전 사이즈 + Android legacy/adaptive/**monochrome**(테마 아이콘) 생성. adaptive 배경 `#F4770C`.
- `dart run flutter_native_splash:create` → 라이트 `#F7F6F3` / 다크 `#131B2A`, Android 12 별도 처리(아이콘 배경 오렌지).
- iOS 다크/틴트: `AppIcon.appiconset` 에 `Icon-App-Dark/Tinted-1024x1024@1x.png` 추가 + `Contents.json` 에 `universal/ios` + `appearances(luminosity: dark|tinted)` 엔트리 수기 등록.

## 2. 릴리스 설정

### iOS
- **ATS 조건화(별도 plist 처리)**: 소스 `Info.plist` 에서 `NSAppTransportSecurity` 제거(Release/App Store = HTTPS 강제).
  - Debug 빌드에서만 로컬 HTTP 허용을 위해 **"Inject Debug ATS" 빌드 페이즈**(pbxproj, `PlistBuddy`)가 산출물 Info.plist 에 `NSAllowsArbitraryLoads=YES` 주입. `CONFIGURATION=Debug` 게이트.
  - (xcconfig 변수 치환 `<${VAR}/>` 은 flutter 툴의 `plutil` Info.plist 파싱을 깨뜨려 폐기 → 빌드 페이즈 방식 채택.)
- 표시 이름 `CFBundleDisplayName` = **작업온** (기존 "Workon").
- 권한 문구(한국어)는 기존 값 유지·확인: 카메라/사진/위치 모두 사유 명시(위치는 "거부해도 사용 가능").
- 버전 1.0.0(빌드 1) 확인.

### Android
- `applicationId` = `kr.workon.workon` 확인, `android:label` = **작업온**.
- **릴리스 서명 골격**: `key.properties` 존재 시 그 키로 서명, 없으면 debug 키 폴백(`flutter run --release` 유지). 템플릿 `android/key.properties.example` 추가. `.gitignore` 가 이미 `key.properties`·`*.jks`·`*.keystore` 제외 확인.
- **minify**: release `isMinifyEnabled=true` + `isShrinkResources=true` + `proguard-rules.pro`(Flutter/Firebase/Kakao/home_widget keep).
- **권한 정리**: main manifest 에 `INTERNET`·`ACCESS_NETWORK_STATE`·`POST_NOTIFICATIONS` 명시(기존엔 debug/profile 에만 INTERNET → **release 무네트워크 버그 수정**). 위치/카메라/사진은 플러그인 병합.
