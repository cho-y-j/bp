# P4 문자 연동 (2026-07-17, app/ 만 수정)

설계 확정본(`worklog/2026-07-17-bizconnect-분석.md`)대로 구현. 기기 기본 문자앱 호출 + 프리필 방식(자동 발송·기본 문자앱 등록 없음, 정책 무리스크). backend·web·live-db·사용자 스택 무접촉.

## 구현 범위

### 1. SmsComposer 브릿지 (MethodChannel `kr.workon/sms`)
- **Dart** `lib/core/sms_composer.dart` — `SmsComposer`(canSendText/canSendAttachments/compose) + `PlatformSmsComposer`. 채널 실패/미지원 시 폴백: 이미지 첨부는 시스템 공유 시트(share_plus `shareXFiles`), 텍스트는 `sms:` 스킴(url_launcher), 최후 공유. Riverpod `smsComposerProvider` 로 주입(테스트 override).
- **iOS** `ios/Runner/SmsComposerPlugin.swift` — `MFMessageComposeViewController`(recipients/body/`addAttachmentData` 이미지·PDF), SceneDelegate 환경 topVC 탐색, delegate 완료 시 dismiss. `canSendText()==false`(시뮬 등)면 `unsupported` 반환→Dart 폴백. `AppDelegate.didInitializeImplicitFlutterEngine` 에서 등록. pbxproj 에 소스 추가(`ios/add_sms_plugin.rb`, xcodeproj gem=CocoaPods 번들).
- **Android** `android/.../SmsComposerPlugin.kt` + `MainActivity.configureFlutterEngine` 등록 — 텍스트: `ACTION_SENDTO smsto:+sms_body`(기본 문자앱 setPackage, 실패 시 패키지 무제한 재시도). 이미지/PDF: `ACTION_SEND(_MULTIPLE)` image/·application/pdf + FileProvider URI + `EXTRA_TEXT` + 기본 문자앱 우선 → 실패 시 `createChooser`(일반 공유) 폴백. `AndroidManifest` FileProvider(`${applicationId}.fileprovider`, `res/xml/sms_file_paths.xml`) + `<queries>`(smsto/tel/SEND).
- **이미지 전처리** `lib/core/image_compress.dart`(순수 Dart, `image` 패키지) — bizconnect `compressImageWithExif`(:457) 참고 재구현: EXIF `bakeOrientation`, 긴 변 ≤1024px, JPEG 품질 반복 하향 목표 ≤300KB. 단위 테스트로 크기 검증.

### 2. "문자로 보내기" 버튼 통합
- **확인서 상세**(`confirmation_detail_screen.dart`): 하단에 [문자로 보내기] + (수기 상대 전화가 있으면) [전화 걸기]. 본문 `smsConfBodyNamed/Plain`("○○님, {현장} 작업확인서 서명 부탁드립니다: {링크}"), 수신인=`extractPhone(conf.contact)` 자동, 링크=`{origin}/c/{shareToken}`. 카톡 공유(기존 재전송)는 유지.
- **QR 명함**(`qr_card_screen.dart`): 공유 버튼 옆 [문자로 보내기](명함 링크).
- **서류 묶음 공유**(`my_shares_screen.dart`): 활성 공유마다 [문자로 보내기](`/s/{token}`).
- 공통 헬퍼 `lib/features/sms/sms_share.dart`(`composeSms` 결과 스낵바, `launchCallAndRecord` tel:+통화기록, `extractPhone`, `confirmationUrl`).

### 3. 빠른 전송 템플릿 (더보기 → "빠른 보내기")
- `lib/features/sms/quick_send_screen.dart` — 기본 3종: **명함**(QR 링크), **사업자등록증**·**통장사본**(서류 지갑에서 유형 키워드 매칭, 마스킹본 우선). 탭 → 수신인 시트(전화 입력 + 전화 있는 팀원 빠른 선택 + `이미지로 첨부` 토글) → 링크/첨부 해석 → 문자 작성창 프리필.
  - 링크 방식: `document-shares` 7일 공유 링크 생성(공개 열람은 마스킹본 제공=개인정보 안전).
  - **이미지로 첨부**: 마스킹본 있으면 마스킹본(백엔드상 PDF)을 첨부(원본 유출 방지), 마스킹 없는 이미지 서류는 원본→`compressForMms`(1024px/300KB)→jpg, 그 외 정규화 PDF.
- **커스텀 템플릿**(`sms_templates.dart`, shared_preferences): 제목·본문·연결(명함/서류)·서류유형. 편집기 시트.
- **변수 치환** `lib/core/sms_template.dart` — bizconnect `TemplateEngine.kt` 참고 Dart 재구현. `{내이름} {상대명} {날짜} {현장} {링크} {시간} {요일}`, 미지원 변수는 원문 유지. 단위 테스트 7건. 기본 템플릿 본문은 l10n 실값(플레이스홀더), 커스텀은 엔진 치환.

### 4. 통화 후 보내기 카드
- `lib/core/call_log.dart` — `CallLogController`(StateNotifier+WidgetsBindingObserver, app_lock 패턴). `recordCall`(tel: 걸 때 상대·시각, shared_preferences 저장), 복귀(resumed) 시 최근 **10분** 내 미제안 통화면 제안. **같은 통화 1회**, 설정 OFF 시 미노출·즉시 닫힘. main.dart 초기값 override(재시작 후 10분 내면 복원). 단위 테스트 6건.
- `lib/widgets/post_call_card.dart` — `main_shell` 상단(모든 탭)에 노출. "방금 ○○님과 통화하셨나요?" [명함 보내기](카드 링크 문자)[빠른 보내기](수신인 프리필로 QuickSend)[닫기].

## 한계 · 미완
- **문자 작성창 실렌더는 실기기 필요**: iOS 시뮬은 `MFMessageComposeViewController.canSendText()==false` 라 네이티브 작성창이 뜨지 않는다(Dart 폴백 경로). 그래서 프리필은 FakeComposer 로 인자(수신인·본문·첨부)를 캡처해 실측했다. Android 에뮬레이터 문자앱·이미지 첨부 실렌더도 실기기 QA 권장.
- **연결 목록/작업 상세 전화 버튼**: `ConnectionItem`·`JobItem` 모델에 전화번호가 없어 해당 화면 직접 전화 버튼은 부착하지 못함(수기 상대=확인서 상세, 팀원=전화 보유분만 부착). 필요 시 백엔드에 연결 상대 전화 노출 추가가 선행돼야 함(백로그).
- **마스킹본은 PDF**: 백엔드 `variant=masked` 는 PDF 반환이므로 "이미지로 첨부"에서 마스킹 서류는 마스킹 PDF 를 첨부(원본 이미지 유출 방지). 이미지 압축(1024/300KB)은 마스킹 없는 원본 이미지에만 적용.
- 원어민 번역 검수 권장(신규 6언어 문구).
