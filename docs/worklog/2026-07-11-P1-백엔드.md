# P1 백엔드 구현 (2026-07-11)

전략 문서(`docs/STRATEGY.md`) P1 항목 중 백엔드 몫: **공수 단위 지원 / 세금계산서 1단계 / 알림톡 어댑터 / 카카오 로그인 마무리**.

## 무엇을 · 왜 · 어떻게

### 1. 공수(품) 단위 지원
- **왜**: 일용직 표준 언어(0.5/1.5공수). 하루 단가 × 공수(소수).
- `RateType` 에 `GONGSU` 추가(마이그레이션). 확인서 단가유형에 `GONGSU` 허용.
- `amount.util.ts`: 기본항목에 `unit?`(공수는 '공수') 추가 — 계산은 기존과 동일하게 `단가 × 수량`, 정수 원 반올림(`money()`). 소수 수량은 기존에도 통과됐고 GONGSU도 동일 경로.
- **소수 검증**: `validateGongsuQuantity()` — 0 초과 + 0.1 단위(0.5 권장). 위반 시 `resolveQuantity()` 에서 400(`INVALID_GONGSU_QUANTITY`). 다른 단가유형은 기존과 동일(정수 강제 없음, 그대로 사용).
- **PDF 라벨**: `confirmations.service.buildPdf` 에서 unit 있으면 `"1.5공수 × 180,000"`, 없으면 기존 `"단가 × 수량"`. 매퍼 `RATE_TYPE_LABEL.GONGSU='공수'`.
- **월 집계**: `ledger.summary` 에 `totalGongsu` 추가 — 그 달 GONGSU 확인서 기본항목 공수 합계(0.1 단위 반올림).

### 2. 세금계산서 1단계 (홈택스 입력용)
- **왜**: 정산 확정 → 홈택스 세금계산서 작성 데이터 자동 정리.
- **공급자(나)**: `profiles` 에 `bizNumber/bizName/bizAddress` 추가(마이그레이션), `PATCH /me` 로 등록. `ProfileDto` 에 노출.
- `GET /ledger/tax-invoice-data?month=&businessId?=`: SIGNED + 미발행(`taxInvoicedAt` null) 확인서만, 상대별 그룹. 공급가액=확인서 `amountCalc.subtotal`, 세액=10%(반올림), 합계. JSON + **복사용 텍스트**(`tax-invoice.util.ts` 순수 함수). `supplierReady`(bizNumber 유무)로 등록 유도.
- **발행 표시**: `ledger_entries.taxInvoicedAt` 추가. `POST /ledger/tax-invoice-data/mark` (`{ ledgerIds }`) → 소유 검증 후 최초 1회 표시(재표시 방지, `marked/alreadyMarked` 반환). 이후 집계 제외.

### 3. 알림톡 어댑터 구조 (발송은 키 대기)
- **왜**: 미가입 상대 도달(카톡 공유보다 강함). FCM/KMA 패턴과 동일하게 "키 없으면 비활성 로그만".
- `notifications/alimtalk/`: `AlimtalkService` 인터페이스 + `ALIMTALK_SERVICE` 토큰 + `SolapiAdapter`(Solapi HMAC 실호출 코드). `SOLAPI_API_KEY/SECRET`·`ALIMTALK_SENDER`·`ALIMTALK_PFID` 모두 있어야 활성. 템플릿 승인 `templateId` 는 `ALIMTALK_TEMPLATE_*` 로 주입(없으면 해당 템플릿 스킵).
- **템플릿 문구 상수**(`alimtalk.templates.ts`): CONFIRMATION_SIGN / PAYMENT_DUE / HEAT_ALERT. 주석에 **카카오 비즈메시지 템플릿 승인 필요** 명시.
- **발송 지점**:
  - 확인서 `send`: 수기 상대(`manualContact`) 있으면 알림톡으로 서명 링크(`sendExternalAlimtalk`, Notification 레코드 없음). 응답에 `alimtalkSent` 추가.
  - 수금 D-day(`LedgerDueScheduler`) / 폭염(`SafetyService`): `notifications.create` 에 `alimtalk` 채널 추가. 정책 **"미가입(푸시 미도달) 상대 전용 우선"** — 푸시가 도달하지 못했을 때만 알림톡 fallback(가입자 중복 발송 방지). 카카오 임시전화(`kakao:`)는 제외.

### 4. 카카오 로그인 마무리
- 기존 `/auth/kakao` 실호출 점검(`fetchKakaoId` → kakaoId 매칭/생성). `kakaoEnabled()` 헬퍼로 정리.
- **`POST /auth/kakao/link`**(인증 필요): 로그인 상태에서 카카오 토큰 제출 → 기존(전화 인증) 계정에 `kakaoId` 연결. 멱등(같은 kakaoId 재연결), 충돌 시 409(`KAKAO_ALREADY_LINKED`: 내 계정에 다른 카카오/또는 이 카카오가 타 프로필에 연결). `KAKAO_ENABLED=false` 면 501.
