# 생체/PIN 앱 잠금 + 카카오 로그인 네이티브 준비 (app 전용)

작성일: 2026-07-17
범위: **app/ 만 수정** (backend·web 무접촉 — 타 에이전트 작업 중)
포트 규칙: 검증은 별도 시뮬레이터에서만 수행(백엔드 미기동 — 잠금은 로컬 게이트라 백엔드 불필요).
사용자 테스트 스택(5432/5439/3010/3001)·live-db 무접촉. 사용자가 테스트 중인 **booted 기본 기기(iPhone 17, 6C713E0F)는 점유하지 않고**, 별도 시뮬(`workon-lock-verify`, iOS 26.5)을 생성해 검증 후 삭제.

## 1. 생체/PIN 앱 잠금 (local_auth)

### 무엇을 / 어떻게
- 의존성 `local_auth: ^2.3.0` 추가.
- **상태 머신** `lib/core/app_lock.dart` — `AppLockController(StateNotifier)`:
  - `AppLockState{enabled, status(locked/unlocked)}`, `isLocked = enabled && locked`.
  - **앱 시작**: 잠금 ON 이면 `locked` 로 초기화(콜드 스타트 잠금).
  - **백그라운드 30초 이상 후 복귀 재잠금**: `WidgetsBindingObserver.didChangeAppLifecycleState` 에서 paused/hidden 시각 기록 → resumed 시 경과 ≥ `backgroundLockThreshold(30s)` 면 재잠금. 라이프사이클 로직은 `onBackground(at)/onForeground(at)` 로 분리해 단위 테스트에서 시각을 주입.
  - `authenticate(reason)` 성공 → 해제 / `setEnabled()` / `markUnlocked()`(명시적 로그인 시).
  - **리프레시 토큰 자동연장과 독립** — 네트워크가 아닌 순수 로컬 게이트.
  - 인증기 추상화 `LockAuthenticator`(→ `LocalAuthenticator` = local_auth). `isDeviceSupported()` 로 **생체 미등록이어도 기기 암호가 있으면 사용 가능** 판정, `authenticate(biometricOnly:false, stickyAuth:true)` 로 **생체 실패/미지원 시 기기 암호 폴백**.
  - `lockAuthenticatorProvider`(override 가능)·`appLockInitialEnabledProvider`(main 에서 override).
- **잠금 게이트/화면** `lib/features/auth/lock_screen.dart`:
  - `AppLockGate` — `MaterialApp.router` 의 `builder` 에서 앱 전체를 Stack 오버레이. `잠금 && 인증됨` 일 때만 `LockScreen` 노출(보호할 세션이 있을 때만).
  - **핵심 게이트 규칙**: `unauthenticated → authenticated`(명시적 로그인)일 때만 `markUnlocked`. **`unknown → authenticated`(앱 시작 세션 복원)은 제외** → 콜드 스타트 잠금이 유지된다. (초기 구현은 이 구분을 안 해 세션 복원 시 잠금이 즉시 풀리는 버그가 시뮬 검증에서 잡혀 수정.)
  - `LockScreen` — 브랜드(오렌지 자물쇠) 아이콘 + "작업온" + "잠겨 있어요" + **인증하고 계속하기**(진입 시 자동 1회 인증, 실패/취소 시 재시도 화면 유지) + **로그아웃**(forceLogout → 게이트 자동 해제, 홈위젯 로그아웃 상태 push).
- **설정 UI** — 더보기 > 설정에 `_AppLockTile`(스위치, 기본 OFF). 켤 때 `canUseDeviceAuth()` 확인 후 미지원이면 안내 스낵바(`appLockUnavailable`)로 켜지 않음.
- **main.dart** — 콜드 스타트 시점부터 게이트가 동작하도록 `SharedPreferences` 로 잠금 초기값을 먼저 읽어 `appLockInitialEnabledProvider` override. `ProviderScopeApp` 이 `overrides` 를 받도록 확장(통합 테스트 재사용).
- **신규 ARB 6키 × 6언어**: `appLockTitle`, `appLockSub`, `appLockLockedTitle`, `appLockUnlock`, `appLockReason`, `appLockUnavailable` (ko/en/zh/vi/ru/ne). 로그아웃/재시도는 기존 키 재사용.
- **iOS**: `Info.plist` 에 `NSFaceIDUsageDescription`(한국어) 추가.
- **Android**: `USE_BIOMETRIC` 권한 + `MainActivity : FlutterFragmentActivity`(local_auth BiometricPrompt 요건).

## 2. 카카오 로그인 네이티브 준비 (키 없이 완결 가능한 전부)

- **iOS** `Info.plist`: `CFBundleURLTypes` 에 스킴 `kakao$(KAKAO_APP_KEY)`, `LSApplicationQueriesSchemes`(kakaokompassauth/kakaolink/kakaoplus/kakaotalk). **빌드 설정 `KAKAO_APP_KEY`** 를 `ios/Flutter/{Debug,Release}.xcconfig` 에 **빈 값 기본**으로 선언 → 키 없으면 스킴이 `kakao`(무해), 키 주입 시 `kakao{앱키}` 자동 완성.
- **Android**: `build.gradle.kts` `manifestPlaceholders["KAKAO_APP_KEY"]` = Gradle 프로퍼티(없으면 빈 값). `AndroidManifest.xml` 에 `com.kakao.sdk.flutter.auth.AuthCodeHandlerActivity` redirect intent-filter(`scheme="kakao${KAKAO_APP_KEY}" host="oauth"`, 키 없으면 no-op) + `com.kakao.talk` queries(플러그인 매니페스트 병합에 더해 명시).
- **dart-define 단일 체인 점검**: `Env.kakaoAppKey`(String.fromEnvironment) 하나로 → 로그인 화면 카카오 버튼 노출(`Env.kakaoEnabled`) + 더보기 카카오 연결 타일 + `KakaoAuth.ensureInit`(`KakaoSdk.init(nativeAppKey)`) + 로그인/연결 호출까지 **런타임 전 체인**이 켜짐을 코드 경로로 확인. 키 미주입 시 전 UI 비노출(전화 인증만).
- **DEPLOY.md 6절 갱신**: "카카오 키 최종 주입 위치" 표 신설 — **① 앱 빌드 dart-define, ② 백엔드 env** 2군데. 네이티브 스킴은 빌드 변수로 자동 완성(app-to-app 까지 쓰려면 릴리스 시 동일 키를 네이티브 빌드 변수에도 미러; 안 넣어도 웹 로그인 동작·빌드 무손상). 5-1 에 네이티브 앱 키/REST 키 구분·플랫폼 등록 추가.

## 한계 / 미완

- **실제 생체(FaceID) 시뮬 조작 미수행**: iOS 시뮬은 기본적으로 기기 암호/FaceID 미설정이라 `isDeviceSupported()`·실제 BiometricPrompt 흐름이 결정적이지 않다. CLI 등록(notifyutil) 신뢰성이 버전별로 불안정해, 검증은 **인증기를 override 한 결정적 게이트 검증**(잠금 화면 표시 → 인증 성공 시 진입 / OFF 시 바로 진입)으로 대체. 상태 머신(30초 타이머·시작 잠금·OFF 통과)은 단위 테스트로 커버. **실기기 생체 인증·기기암호 폴백·백그라운드 30초 재잠금 UX 는 실기기 QA 권장.**
- 카카오 실연동은 백엔드 스텁(501) 상태(리프레시 worklog·기존 백로그와 동일). 네이티브/앱 준비는 이번에 완결, REST 키 실연동은 백엔드 후속.
- app-to-app(카카오톡 전환)까지 활성화하려면 릴리스 빌드 시 네이티브 빌드 변수에도 동일 키 필요(DEPLOY 6절 명시). 없어도 카카오계정 웹 로그인·빌드는 정상.
