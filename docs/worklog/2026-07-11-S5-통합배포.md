### 발견·수정한 문제 (운영 구성에서만 드러남)
1. **[중요] 웹 API 베이스 오설정** (`web/lib/api.ts`): `NEXT_PUBLIC_API_URL` 하나를 브라우저·SSR 양쪽에 쓰고
   기본값이 `http://localhost:3010/api` → 운영 빌드에서 깨짐.
   → **브라우저용은 상대 `/api`**(nginx 동일 오리진, HTTPS 그대로), **SSR 용은 `API_INTERNAL_URL`(=`http://api:3000/api`)** 로 분리.
   `absoluteUrl` 도 상대 base 면 상대경로 반환 → 파일/PDF 링크가 브라우저 오리진으로 해석(playwright 로 확인).
   compose web 서비스에 `API_INTERNAL_URL` 추가.
2. **[중요] Next.js standalone 헬스체크 실패** (`web/Dockerfile`): standalone `server.js` 는 `$HOSTNAME` 에 바인딩하는데
   Docker 가 HOSTNAME 을 컨테이너 ID 로 설정 → 127.0.0.1 로 접속 불가 → healthcheck unhealthy → 의존 nginx 미기동.
   → `ENV HOSTNAME=0.0.0.0` 추가. (127.0.0.1/0.0.0.0 둘 다 ECONNREFUSED 재현으로 원인 확정)
3. **[중요] 공개 링크 도메인 누락**: `PUBLIC_WEB_URL`(카톡 공유 링크에 박힘)·`GOV_DATA_SERVICE_KEY`·`KAKAO_ENABLED` 가
   compose api env 에 미전달 → 운영에서 공유 링크가 `localhost` 로 굳음.
   → base compose api env 에 3종 추가 + `.env.example` 정비.
- 각 수정 후 **fresh 볼륨으로 재빌드·재기동하여 시나리오 ①~⑤ 재검증 통과**.
