# S2b 서류·장비 도메인 (2026-07-11)

## 구현 범위
S2a(인증/공통 인프라) 위에 서류·장비 도메인을 구현했다.

### 스키마 (마이그레이션 `20260711020000_s2b_documents_equipment`)
- **`_SharedDocuments` 암묵 M:N → 명시적 조인 모델 `SharedDocument`** 승격
  (documentShareId, documentId, `useOriginal Boolean @default(false)`, @@unique 조합).
  → 서류별 원본/마스킹 선택을 per-doc 플래그로 관리.
- Document 보강: `originalFilePath`, `originalFileName`, `fileSize`, `mimeType` (원본 보존/메타).
- DocumentShare 보강: `revokedAt` (즉시 무효화).
- ⚠️ 마이그레이션 폴더명 주의: 생성 시각(머신 시계)이 S2a보다 빨라 폴더명을
  `20260711020000_...` 로 **수동 리네임**해 배포 순서(init→s2a→s2b)를 보장.
  fresh `migrate deploy` 로 순서 검증 완료.

### /equipments (본인 소유만)
- CRUD (`equipments.service/controller/module`). 소유자 불일치는 404(존재 은닉).
- 삭제 시 연결 서류는 DB onDelete Cascade.

### /documents
- `POST /documents` (multipart, 최대 20MB, memoryStorage): jpg/png/heic/webp/pdf 허용.
  - **이미지 → PDF 정규화** (`PdfService.normalizeToPdf`, sharp `.rotate()` EXIF 보정 → PNG → pdf-lib 단일 페이지). HEIC/HEIF 는 `heic-convert` 로 JPEG 선변환.
  - **원본 보존**: `uploads/{profileId}/{documentId}/original.{ext}` + `normalized.pdf`.
  - 경로 조작 방지: `FileStorageService.safeResolve`(루트 밖 차단) + id 정규식 화이트리스트 + 고정 파일명.
- `GET /documents` (D-day 포함, `?groupByEquipment=true` 그룹) / `PATCH /:id` / `DELETE /:id`(파일 디렉터리 삭제).
- `POST /documents/:id/mask`: 페이지별 정규화(0~1, 좌상단 원점) 사각형 → PDF 좌하단 원점 변환(`normalizedRectToPdf`, 클램프) → 검정 사각형 오버레이 → `masked.pdf` 저장, `maskedFilePath` 기록.
- `POST /documents/:id/verify`: 사업자등록증 → `BizVerifyService`(국세청 odcloud validate, `GOV_DATA_SERVICE_KEY` 없으면 501·있으면 실호출). 그 외 유형 → `{result:'UNSUPPORTED', manualCheck:true}` + 상태 유지. 결과는 `verificationResult` JSONB.
- `GET /documents/expiring?days=30`.
- D-day 계산은 **KST 자정 기준** 순수 함수(`common/dday.util`) — 서버 TZ 무관.

### /document-shares (선택 묶음 공유)
- `POST /document-shares`: 전부 본인 서류 검증 → `SharedDocument` 조인 생성. `expiresInDays`(기본7·최대30), `perDocument[].useOriginal`. share_token `nanoid(32)`. `url = {PUBLIC_WEB_URL}/s/{token}`.
- `GET /document-shares`(열람 로그 포함) / `DELETE /:id`(revokedAt 설정, 즉시 무효화).
- `GET /public/shares/:token` (@Public): 만료/무효화 검증 → 메타 목록, viewLogs 에 `{at,ip,ua}` append (X-Forwarded-For 고려).
- `GET /public/shares/:token/files/:documentId` (@Public): 정책 적용 파일 스트림 (useOriginal→정규화본, 아니면 마스킹본 우선). Content-Disposition RFC5987 한글 파일명.

### 만료 알림 준비
- `NotificationsService` 스텁(Notification 레코드 생성 + `sendPush` 로그 스텁, FCM 은 S2d).
- `@nestjs/schedule` 크론 `DocumentExpiryScheduler` (매일 09:00): D-30/7/0 대상 → 알림 생성. 같은 서류·D-day 중복 방지.

## 검증 (실제 실행)
- **lint**: 0 errors (test 파일의 no-unsafe-argument 경고 30건은 기존 auth e2e 와 동일 성격).
- **build**: 통과 (`nest build`).
- **단위 37/37 통과**: dday(KST/경계), 마스킹 좌표 변환(y반전·클램프), 공유 만료/권한/마스킹정책, 스케줄러 D-30/7/0·중복방지. (기존 auth 9 포함)
- **e2e 13/13 통과** (임시 pg 5433): 업로드→PDF변환 확인→마스킹→묶음공유→public 열람→로그 기록→만료 403/무효화 403, 진위확인 UNSUPPORTED/501, 장비 서류 그룹.
- **실서버 curl 실검증** (port 3010, temp pg 5433, ImageMagick 부재로 sharp 로 실제 JPEG/PNG 생성):
  - JPEG 업로드 → `normalized.pdf` 1페이지 600x380pts (pdfinfo 확인), `original.jpg` 보존.
  - 마스킹 2영역 → masked.pdf 1페이지, 크기 5462→5865B 증가(사각형 반영).
  - 공유 → 32자 토큰 + `/s/{token}` URL. public 열람(무인증) masked=true.
  - public 파일 다운로드 → `application/pdf`, `filename*=UTF-8''%EC%8B%A0%EB%B6%84%EC%A6%9D.pdf`, 크기=masked(5865) → 마스킹 정책 적용 확인.
  - 열람 로그: viewCount 1, ua=curl-verify-agent, ip=127.0.0.1.
  - 만료(expiresAt 과거) → 403 SHARE_EXPIRED / DELETE 후 → 403 SHARE_REVOKED.
  - 사업자등록증 verify(키 없음) → 501 / 신분증 verify → UNSUPPORTED / txt 업로드 → 400 / DELETE → 파일 디렉터리 제거.
- 임시 pg 컨테이너·uploads·live 서버 정리 완료. **live-db(5432) 무손상 확인**.

- **파일 저장 규약 재사용**: `FileStorageService`(경로 조작 방지, `uploads/{profileId}/{id}/`)와 `PdfService` 는 확인서 PDF/서명 이미지에도 재사용 가능(이미 exports). 서명 이미지 base64 저장 시 같은 저장소 패턴 사용 권장.
