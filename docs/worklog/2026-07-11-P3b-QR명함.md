# P3b — QR 명함 (작업자 공개 프로필) (2026-07-11)

> STRATEGY.md P3-신규 3 구현. 오프라인 확산 루프: 작업자 공개 프로필 QR → 웹 페이지(이름·업종·서류 유효 배지·장비·연결 CTA).
> 포트: 임시 pg 5436 · 백엔드 3040 · web 3004 만 사용. live-db(5432) 무손상.
> P3a 이월 준수: 공개 경로 `/p/{token}` 별도(확인서 `/c/{shareToken}` 와 분리), 계좌·전화·서류 파일 절대 비노출.

## 1. 무엇을 / 왜
작업자가 자기 QR 명함을 오프라인(현장·명함)에서 뿌리면, 상대가 스캔 → 로그인 없이 공개 프로필 웹(`/p/{token}`)을 보고 "앱에서 전화번호로 검색해 연결"로 유입. 신뢰 신호로 **서류 유효 배지**(민감정보 없이 "유효한 서류를 갖췄다"는 사실만)를 노출. 부정 정보·민감정보는 일절 비노출.

## 2. 백엔드 (구현·검증 완료)

### 스키마 (마이그레이션 `20260711110000_p3b_qr_card`, additive)
- `Profile`: `cardToken String? @unique`(nanoid 32, 온보딩 후 최초 `GET /me/card` 시 lazy 생성), `cardEnabled Boolean @default(true)`, `cardIntro String?`(한 줄 소개), `cardViewCount Int @default(0)`(공개 조회수 — 로그 최소화, 카운터만). unique index `profiles_cardToken_key`.

### 엔드포인트 (신규 `card` 모듈)
- `GET /me/card`(인증) — 토큰 lazy 생성 + `url`(`{PUBLIC_WEB_URL}/p/{token}`) + `preview`(공개 프로필과 동일) + **`docStatus.expiredDocs`(본인 전용 — 만료된 내 서류 유형·만료일·D-day)**.
- `POST /me/card/rotate`(인증) — 토큰 재발급(유출 대비, 유일성 충돌 재시도). 구 토큰 즉시 무효화.
- `PATCH /me` — `cardEnabled`·`cardIntro` additive(UsersService/UpdateMeDto/ProfileDto 확장).
- `GET /public/profiles/:token`(**@Public**) — 이름, 업종 태그, 한 줄 소개, **서류 유효 배지**(만료일 등록 ≥1 && 만료 지난 것 0 → valid, 유형명·개수만), 장비 목록(**종류만**), 가입일, 연결 유도(`connect`: 앱 검색 안내 문구 + 딥링크/스토어 링크 자리). **전화·계좌·서류 파일/경로·발급일 절대 비노출**. `cardViewCount`만 증가. cardEnabled=false·무효 토큰 → 404 PROFILE_CARD_NOT_FOUND.

### 서류 유효 판정 (순수 함수 `card.util.ts` — `computeDocValidity`)
- ARCHIVED 제외. `valid = (만료일 등록 서류 ≥1) && (만료 지난 서류 0)`. `computeDday`(KST 자정 기준) 재사용, D-0(오늘 만료)은 아직 유효. 유형명은 dedupe·정렬해 노출.
