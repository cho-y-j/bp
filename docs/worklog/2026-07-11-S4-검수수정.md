# S4 독립 검수 지적 수정 (2026-07-11)

> 대상: `app/` (작업온 Flutter 앱). 파일:라인은 검수 시점 기준.

## 검수 요약 (지적 → 조치)

### 중요
1. **네트워크 오류 재시도** — 에러 표시부가 raw 예외 문자열(`불러오지 못했습니다: $e`)을
   그대로 노출. → 공통 `ErrorRetry` 위젯 신설(`widgets/common.dart`): 친화 메시지
   "연결에 문제가 있어요" + 안내 + **"다시 시도" 버튼**(해당 provider `invalidate`).
   홈(오늘 일정·이번 달 요약), 캘린더, 장부, 회사상세, 지갑, 알림, 사업장 모드,
   작업자, 장비, 수신함 상세 — 에러 표시부 일괄 교체. 홈/장부의 당겨서-새로고침
   `onRefresh` 는 실패를 삼켜(try/catch) 각 섹션 ErrorRetry 로만 표시(미처리 예외 제거).
2. **장부 빈 상태 CTA** — "이 달의 장부 기록이 없어요." 단문만 노출. → `_LedgerEmpty`
   위젯: 홈 빈 상태와 통일된 `PaperCard`("이 달의 장부 기록이 없어요" +
   "확인서를 작성하면 장부가 자동으로 채워져요.") + **확인서 작성 이동 버튼**(`PrimaryButton`).

### 사소
3. **dispose 누락** — `biz_confirmation_detail_screen`(`_sig`,`_nameCtl`),
   `business_mode_screen._CreateFlow`(`_nameCtl`,`_bnoCtl`),
   `workers_screen`(`_phoneCtl`, `_JobForm` `_siteCtl`/`_rateCtl`) State 에 `dispose` 구현.
   바텀시트 임시 컨트롤러: `company_detail` 입금시트 `amountCtl` →
   `showModalBottomSheet(...).whenComplete(amountCtl.dispose)`,
   `equipment_screen` 추가시트 `typeCtl`/`vnoCtl`/`specCtl` → `try/finally` 해제.
4. **수량 0 검증** — `confirmation_form_screen._valid` 에 `quantity > 0` 요구 추가.
   0 이면 저장 버튼 비활성 + 수량 필드 아래 안내("일수/시간/건수를 1 이상 입력해 주세요").
5. **BuildContext / pop-후-SnackBar** —
   - `use_build_context_synchronously` 2건: `biz_confirmation_detail_screen`(_sign, `await _load` 이후),
     `ledger_screen`(명세서 열기 실패 SnackBar) → await 이전 `ScaffoldMessenger` 캡처 + `mounted` 가드.
   - pop-후-SnackBar 2건: `mask_editor`(_save), `confirmation_form_screen`(_submit) →
     await 이전 루트 `ScaffoldMessenger`/`Navigator` 캡처 후 pop→스낵바(공유 시트는 pop 이전 호출).
6. **서명 1MB 검사** — `signature.dart` `encodeSignatureDataUri` 가 base64 **문자열 길이**를
   비교(백엔드보다 과하게 보수적). → 백엔드(`confirmations.service.decodeSignPng`:
   `buf.length > MAX_SIGN_BYTES`)와 동일하게 **디코드 PNG 바이트(`png.length`)** 기준으로 검사.
