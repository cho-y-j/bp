# 2026-07-17 · 웹 작업자 모드 (`/w/*`)

> 작업온 웹(Next.js 15)에 **작업자 모드** 신설. 기존엔 웹이 "사업장 중심"이라 작업자는 앱에서만 확인서·장부·서류·소득을 다뤘는데, 이제 데스크톱/모바일 웹에서도 가능하게 함. **web/ 만 수정**(backend·app 무수정 — 기존 API 계약만 사용).

## 범위·구현

신규 섹션 `/w/*` (biz 레이아웃 패턴·시안 토큰·기존 컴포넌트 재사용, 한국어):

- **로그인 후 분기 개선** (`app/login/page.tsx`): 명시적 `?next=` 없으면 로그인 직후 `/me` 조회 → **사업장 보유 시 `/biz/inbox`, 미보유 시 `/w/home`**. 기존엔 항상 `/biz/inbox`로 가 사업장 미보유자가 "사업장 생성 게이트"에 갇혔음. 게이트는 이제 `/biz` 직접 접근 시에만 노출(biz 레이아웃 유지). biz 레이아웃·게이트에 "작업자 모드" 링크, 작업자 레이아웃에 "사업장 모드/사업장 만들기" 링크 추가(상호 전환).
- **`/w` 레이아웃** (`app/w/layout.tsx`, `w-context.tsx`): `/me` 로드, 사이드바(데스크톱)+상단 탭바(모바일) — biz의 `biz-shell/biz-sidebar/biz-topbar` CSS 그대로 재사용. 탭: 홈·확인서·장부·서류·소득.
- **`/w/home`**: 이번 달 요약(`ledger/summary` — 일한 날·공수·미수·입금) + 최근 확인서 5건(`confirmations?month=`) + 만료 임박 서류(`documents/expiring?days=30`, D-day 배지) + 대기 알림(`notifications?unread=true`).
- **`/w/confirmations`**: 월 선택 목록(상태 배지 작성중/전송됨/서명완료) + **작성 폼**(날짜·현장·상대[연결 사업장 선택 `GET /connections` ACCEPTED or 수기 회사명+연락처]·작업내용·시간·단가유형[일당·공수·시급·건당, **공수 0.5 스텝·0.1 단위 검증**]·장비 섹션 토글[장비명·차량번호·규격·유도원]·실시간 금액 미리보기) + 상세 모달(**PaperConfirmation 재사용**)·`send`(링크 발급+클립보드 복사)·PDF(인증 blob 새 탭)·`revoke`(SENT만 노출).
- **`/w/ledger`**: 월 요약 + 회사별 미수(`ledger/by-company`, D-day 배지·상태) + 개별 항목(`ledger/entries`) — 부분입금 기록(`POST /ledger/:id/payments`)·자동 독촉 토글(`PATCH /ledger/:id {autoRemind}`, 파생 항목 숨김)·수동 독촉(`POST /ledger/:id/remind`) + 월간 명세서 PDF(`ledger/statement`). 팀 파생 항목은 "팀 파생" 배지+읽기전용.
- **`/w/documents`**: 탭(내 서류/공유 관리). 서류 목록(D-day 배지·마스킹본 배지) + 업로드(multipart, 이미지/PDF→백엔드 PDF 정규화) + 체크 선택 → 묶음 공유 링크 생성(`POST /document-shares`, 기본 7일)·복사 + 내 공유 관리(`GET /document-shares`, 열람수·만료·무효화 `DELETE`). **마스킹 편집기는 v1 제외**(캔버스 작업 큼) — "마스킹은 앱에서" 안내 배너 + 백로그.
- **`/w/income`**: 탭(소득 리포트/세금계산서 준비). 리포트: 연도 선택·총계(청구/입금/미수/일수·공수/팀지급분·순소득)·월별 추이 표·상대별 표·종소세 안내·PDF(`ledger/income-report[/pdf]`). 세금계산서: 월 선택·공급자 상태·상대별 그룹(일자·내용·공급가액 표, 공급가/세액)·복사용 텍스트(`ledger/tax-invoice-data`)·발행완료 표시(`.../mark`).

**공용 유틸**: `lib/worker.ts`(`openAuthedPdf` 인증 blob→새 탭·`copyText` 클립보드+폴백), `app/w/types.ts`(응답 타입 미러·배지 헬퍼). `components/Icons.tsx`에 Home/Folder/Coins/Upload/Copy/X/Bell/Refresh/Trash 추가. `app/globals.css`에 `.w-stat-grid`·`.w-tabs/.w-tab`·`.w-mini-table`·`.w-toolbar`·`.w-btn-row`·`.w-table-scroll` 유틸(모바일 반응형 포함) 추가.

## 백로그
- 서류 **마스킹 편집기(캔버스)** 웹 이식(v1은 "앱에서" 안내). 백엔드 `POST /documents/:id/mask` API는 이미 존재.
- 확인서 작성 폼의 **추가 항목(연장·야간 등 additionalItems)·부가세율** 입력(현재 기본 단가×수량만; 백엔드는 지원). 팀 확인서 작성(teamId/teamEntries)도 웹 미노출.
- 공개 페이지 외 **i18n**은 biz 웹과 동일 정책으로 한국어 우선(백로그).
- 소득 세금계산서 **공급자 정보 등록**은 웹 프로필 화면 부재로 "앱에서" 안내(백엔드 `PATCH /me` 지원).
