전략 문서(`docs/STRATEGY.md`) P1 중 **앱 몫**: 공수(품) 입력 / 오프라인 임시저장 / 세금계산서 화면 / 카카오 로그인 버튼(조건 노출). 백엔드 P1은 완료(`worklog/2026-07-11-P1-백엔드.md`) 상태에서 API에 UI를 연결했다.

## 무엇을 · 왜 · 어떻게

### 1. 공수(품) 입력
- **왜**: 일용직 표준 언어(0.5/1.5공수). 하루 단가 × 공수(소수).
- **단가유형 세그먼트**에 `공수` 추가(일당/공수/시급/건당). `lib/features/confirmation/confirmation_form_screen.dart`.
- 공수 선택 시 수량 입력을 **0.5 스텝퍼**(`_GongsuStepper`, 0.5~5.0, −/값/+)로 전환. **직접 입력도 허용** — `validateGongsuQuantity`(0 초과·0.1 단위, 백엔드 `validateGongsuQuantity`와 동일 로직, `lib/core/amount_calc.dart`)로 검증. 위반 시 저장 버튼 비활성 + 인라인 안내, 서버 `400 INVALID_GONGSU_QUANTITY` 도 스낵바로 대응.
- **금액 계산 미리보기**: `calcAmount` 에 `GONGSU` 라벨(`기본(공수)`)·`unit='공수'` 반영 → 미리보기 `180,000 × 1.5공수 = 270,000`.
- **월 요약 totalGongsu 표시**: 홈(`"일한 날 2일 · 1.5공수"`)·장부(`"이번 달 2일 · 1.5공수 일함"`). 공수 0이면 기존대로 일수만. `formatGongsu`/`daysWithGongsu`(`lib/core/format.dart`), `LedgerSummary.totalGongsu`.
- **캘린더 주간·확인서 카드 라벨**: `amountCalc.items[0].unit` 을 읽어(`Confirmation.baseUnit/baseQuantity/baseRate`) `"1.5공수"` 표기. 캘린더 주간 아이템 부제·확인서 상세 단가 줄.

### 2. 오프라인 임시저장 (현장 신호 불량 대비)
- **왜**: 현장 신호 불량 시 확인서 유실 방지 → 연결 복구되면 자동 전송.
- **로컬 초안 큐**(`lib/core/draft_store.dart` — `shared_preferences` JSON 직렬화, 순수 `encodeList/decodeList`). 저장 액션 네트워크 실패 시 `POST /confirmations` 본문을 큐에 enqueue + `"임시저장됨 — 연결되면 자동 전송돼요"` 스낵바.
- **연결 복구 자동 전송**: `connectivity_plus` 스트림 감지 → `DraftQueue.flush`(`lib/providers/drafts.dart`). 성공은 큐 제거·홈/장부 무효화, **서버 검증 실패(4xx)는 초안에 `lastError` 표시 후 유지**(사용자 개입 유도), 네트워크/5xx는 유지 후 재시도. `isRetriableFailure` 로 분류(NETWORK/5xx=재시도, 4xx=초안 유도).
- **홈 대기 배너**(`_DraftsBanner`, `N건 전송 대기`) → **초안 목록**(`lib/features/drafts/draft_list_screen.dart`): 수동 재시도(`지금 모두 전송`)·개별 삭제. 자동 전송 결과는 `MainShell` 전역 스낵바(`draftFlushEventProvider`).
- **작성 폼 이탈 시 자동 초안 1건 보존**(`AutoDraftController`, prefs 단일 슬롯): `PopScope` 로 dirty 상태면 저장, 재진입 시 복원 배너(`불러오기`/`삭제`). 저장·임시저장으로 커밋되면 자동 초안 제거.

### 3. 세금계산서 화면 (더보기 → "세금계산서 준비")
- **프로필 사업자 정보**(`lib/features/tax/business_info_screen.dart`): `bizNumber/bizName/bizAddress` → `PATCH /me`(`AuthController.saveBusinessInfo`). 미등록이면 세금계산서 화면에서 입력 유도(`supplierReady=false`).
- **화면**(`lib/features/tax/tax_invoice_screen.dart`): 월 선택 → `GET /ledger/tax-invoice-data` → 공급자 요약 + 홈택스 안내 + **상대별 카드**(공급가/세액10%/합계/품목 수, `buyerRegistered` → `등록 상대`/`확인 필요` 배지).
- **복사 버튼**: 그룹 텍스트를 클립보드(`taxInvoiceCopyText` 순수 함수, `lib/features/tax/tax_invoice_text.dart`) + 홈택스(hometax.go.kr) 붙여넣기 안내. **발행 완료 표시**(`POST /ledger/tax-invoice-data/mark`) → 목록에서 제외.

### 4. 카카오 로그인 버튼 (조건 노출)
- 의존성 `kakao_flutter_sdk_user` 추가. SDK 래퍼 `lib/core/kakao_auth.dart`(톡 로그인 우선·계정 폴백, 앱 키 있을 때만 init).
- **`--dart-define=KAKAO_APP_KEY` 주입 시에만 노출**(`Env.kakaoEnabled`). 없으면 전화 인증만(기존과 동일). 로그인 화면 카카오 버튼 → `POST /auth/kakao`, `501(NOT_IMPLEMENTED)`이면 `"카카오 로그인 준비 중"` 안내.
- 더보기(설정)에 **카카오 계정 연결**(`kakao/link`, 동일 조건 노출) — `409 KAKAO_ALREADY_LINKED`·`501` 분기 안내, 연결됨 상태 표시.

## 미완 / 다음(P1-다국어 등)에 넘길 주의사항
- **문자열 하드코딩**: 이번 P1 앱 UI는 기존 관행대로 **한국어 문자열이 위젯에 하드코딩**돼 있다(공수/임시저장/세금계산서/카카오 문구 포함). P1-다국어 배치에서 `flutter_localizations`+ARB 이식 시 신규 문구도 함께 추출 필요. 특히:
  - 공수 스텝퍼/검증 문구(`공수는 0.1 단위로…`), 초안 배너·목록·스낵바(`임시저장됨 — 연결되면 자동 전송돼요`, `N건 전송 대기` 등), 세금계산서 화면 전체(홈택스 안내·배지·버튼), 카카오(`카카오로 시작하기`·`카카오 로그인 준비 중`·`카카오 계정 연결`).
  - 순수 포맷 함수(`daysWithGongsu`, `taxInvoiceCopyText`)도 로케일화 대상. `taxInvoiceCopyText`/서버 `text` 는 **한국어 홈택스 입력 포맷 고정**이 맞다(외국어 UI라도 계산서 텍스트는 한국어 유지 권장) — 다국어 배치에서 정책 확정 필요.
- **카카오 실연동 네이티브 설정 미포함**: 앱 키 미주입이라 UI만 조건부 노출. 실제 카카오 로그인 활성화 시 **iOS Info.plist**(`CFBundleURLSchemes: kakao{APP_KEY}`, `LSApplicationQueriesSchemes: kakaokompassauth/kakaolink`)·**Android**(`AuthCodeHandlerActivity` redirect, `KAKAO_APP_KEY` manifest placeholder) 설정과 `KAKAO_ENABLED=true` 백엔드가 필요(DEPLOY.md 6절 카카오 항목과 연동).
- **오프라인 자동 전송 트리거**: `connectivity_plus` 연결 복구 이벤트 기반(단위 테스트로 flush 로직 검증). 시뮬레이터 integration_test는 연결 이벤트를 인위 발생하기 어려워 **수동 재시도 경로로 최종 확인**(자동 flush 로직 자체는 단위 테스트 커버). 실기기 기내모드 토글 시 자동 전송 동작.
- 세금계산서 복사는 **상대별 카드 단위**(그룹 텍스트). 서버가 주는 전체 `text` 는 미사용(그룹 단위가 홈택스 1건 발행에 더 부합) — 필요 시 전체 복사 버튼 추가 여지.
