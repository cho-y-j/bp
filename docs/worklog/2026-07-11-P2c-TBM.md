## 무엇을 / 왜 / 어떻게

### 백엔드 (완료·검증)
1. **스키마(마이그레이션 1개, 순수 additive)** `20260711091745_p2c_tbm`:
   - enum `TbmPresetKind`(HAZARD|MEASURE), `NotificationType`+`TBM`, `SafetyLogType`+`TBM`(ALTER TYPE ADD VALUE — 기존 값 미사용).
   - `tbm_records`: businessId(Cascade), authorProfileId?(SetNull), site, occurredAt, **hazards Json**([{code|text}] 키 기반), measures?, notes?, photoPaths String[], created/updatedAt.
   - `tbm_attendees`: recordId(Cascade), profileId?(SetNull), name(스냅샷/수기), **ackAt?**(최초 1회 UPDATE), createdAt.
   - `tbm_presets`: businessId(Cascade), kind, text(원문). Business/Profile 관계 additive.
2. **tbm 모듈**(`src/tbm/`): 2개 컨트롤러 — `biz/tbm`(사업장), `tbm`(작업자). 서비스 + 매퍼 + DTO 3종 + 공통 프리셋 상수(`common/tbm-presets.ts`).
3. **작성/수정/삭제**(소유자만 `ownedBusinessOrThrow`/`ownedRecordOrThrow`): create 시 참석자 해석(가입=전화검색 동의 or ACCEPTED 연결 검증, 수기=이름 필수). **당일만 수정/삭제**(`isSameKstDay(createdAt, now)` — 이후 409 NOT_EDITABLE/NOT_DELETABLE, 증빙 무결성). update 시 attendees 지정하면 명단 전체 대체(신규 가입자만 추가 알림).
4. **참석자 확인(ack)**: `POST /tbm/:attendeeId/ack` — 본인(profileId)만, **조건부 updateMany(ackAt=null 일 때만) 원자 전이**로 재확인 409 ALREADY_ACKED. 확인 시 safety_log(TBM_ACK, receivedAt/confirmedAt/ackAt) append + 사업장 소유자 알림.
5. **safety_logs 연동**: 기록 생성 → safety_log(type TBM, targetProfileId=작성자, payload.kind=TBM_RECORD) append. 확인 → safety_log(type TBM, targetProfileId=참석자, payload.kind=TBM_ACK, ackAt) append. → 안전 리포트 "발송·확인 기록" 표와 유형별 건수에 자동 반영.
6. **사진**(FileStorageService 재사용): `POST /biz/tbm/:id/photos`(FilesInterceptor·당일만·20MB·이미지 MIME 화이트리스트, `buildKey(businessId, recordId, tbm-photo-N.ext)`), `GET /biz/tbm/:id/photos/:idx`(소유자)·`GET /tbm/:id/photos/:idx`(참석 작업자만).
7. **프리셋**: `/biz/tbm/presets` 간단 CRUD(소유자만). 기본 위험요인 프리셋 10종(중장비 협착·고소작업 추락·폭염 온열질환·감전·낙하물·붕괴·화재폭발·분진소음·전도·밀폐공간)은 **코드 기반**(DB 미저장 — 앱이 6언어 번역), 커스텀만 원문 저장.
8. **위험요인 키 기반**: 저장은 `[{code}|{text}]`. 서버 PDF/리포트(정본 한국어)는 `TBM_HAZARD_LABEL_KO` 로 코드 치환, 앱은 코드로 작업자 자기 언어 렌더.
9. **안전 리포트 TBM 섹션**: `renderSafetyReportPdf` 확장 — biz.service.safetyReport 가 tbm_records 월간 집계(참석 `_count`, 확인 `groupBy ackAt not null`)를 `tbm[]` 로 전달 → PDF 에 "■ TBM(안전점검회의)" 표(일자·현장·위험요인 요약·참석/확인) 렌더(페이지 브레이크 처리).

## 미완 / 다음 배치(P2d 연간 소득 리포트) 주의사항
- **TBM 수기 참석자**: profileId 없는 수기 참석자는 알림·ack 대상 아님(오프라인 서명부 성격). 필요 시 링크/알림톡 확장 여지.
- **참석자 이름 마스킹**: 폼에서 연결 작업자 칩은 `/connections` 의 마스킹명(예 `안**로`)을 스냅샷으로 저장 → 상세·리포트도 마스킹명 표기(앱 전반의 maskName 관례와 일치). 원문 필요 시 create 에서 name 생략하면 백엔드가 profile.name 스냅샷.
- **P2d 주의**: ①소득 집계는 ledger_entries(payments/미수) + confirmations(팀 파생 포함) 기준 — P2a 파생 항목 중복 합산 주의(반장 합계 vs 팀원 파생). ②월별 추이/상대별/공수는 기존 `/ledger/by-company`·`kstMonthRange` 재사용. ③PDF 는 `renderStatementPdf` 확장 또는 신규 renderer — 나눔고딕·페이지 브레이크 패턴 재사용. ④세금계산서(P1 taxInvoicedAt) 제외분 처리 로직 재사용 가능. ⑤안전 리포트처럼 `GET /ledger/income-report?year=` 형태 인증 blob PDF + 앱 더보기 진입.
- **웹**: TBM 은 앱 전용(웹 UI 없음 — 설계상 사업장 현장 기록은 모바일). 필요 시 P2 통합에서.
