## 항목별 개선

### 1. 종이 확인서 서명 — 도장(인영) 박스 렌더 [★2]
- **공개 API 제약 확인(중요)**: `GET /public/confirmations/:token`(`confirmations.service.ts publicView`)와 사업장 수신 상세(`biz.service`)는 **서명 이미지 데이터/경로를 반환하지 않는다**(`signerName`·`signedAt` 만). 내부 `signImagePath` 는 매퍼에서 의도적으로 비노출(파일 서빙 공개 라우트 없음 — 컨트롤러 `:token`·`:token/pdf`·`:token/sign` 만). PDF 는 서버가 임베드하지만 화면용 이미지 접근 경로가 없다. → **손글씨 서명 이미지 실렌더는 백엔드 수정 필요**(아래 백엔드 요청사항).
- **web-only 개선**: 기존 "✓ 박사장 님 서명 완료" 초록 텍스트 배지를 **도장(인영) 박스**로 교체 — 붉은 인주색 원형 인영(서명자명 각인, 살짝 기울임·이중 테두리)에 서명자명 큰 글씨 + "서명 완료" ✓ + 서명 일시. 종이 확인서의 시각적 클라이맥스로 승격. 6언어 라벨 대응(`paperSignConfirmed` 키 신설).
- 파일: `components/PaperConfirmation.tsx`(`sealText()`·sign-plate), `app/globals.css`(`.sign-plate`·`.seal`·`--seal` 토큰 라이트/다크), 6개 사전 `paperSignConfirmed`.

### 2. 다국어 금액 — ₩ + 한국식 콤마 고정 [★8]
- `lib/format.ts money()` 를 로케일 그룹핑(`grouped()`, ru 공백·일부 로케일 점)에서 **항상 `won()`(ko-KR 콤마)** 로 일원화. 한국어 `1,050,000원`, 그 외 `₩1,050,000`. `grouped()` 폐기.
- **검증(6언어)**: `₩1.050.000`(vi, 점 오독) → `₩1,050,000`. ru `₩1 050 000`(공백) → `₩1,050,000`. ko/en/ne/zh 전부 콤마 확인.

### 3-a. QR 명함(/p) — 종이 명함 메타포 리디자인 [★7]
- 제네릭 카드 → **종이 명함**: 상단 오렌지 그라데이션 헤더(브랜드·"작업자 명함"), 큰 이름, 업종 역할 부제, 소개, **"서류 유효" 인증 스트립 강조**(초록·방패+체크, 감사 지적 "서류 유효 배지 강조"), 서류 유형·장비(트럭 아이콘) 칩. 데스크톱·모바일 반응형. `app/p/[token]/page.tsx`·`.namecard*` CSS.

### 3-b. 랜딩(/) — 한 문장 + 큰 CTA 1개 단순화 [★7]

### 4. biz 화면 정리 + D-day·상태 앱 규칙 통일 [★9]
- **죽은 공간**: `.biz-main` max-width 1000→**880px**(행이 과도하게 늘어나 이름↔금액 시선이동이 길던 문제 완화). 표/행 hover 하이라이트(`.data-table tr:hover`·`button.row-item:hover`)로 스캔성 강화.
- **D-day 통일**(`lib/format.ts`): `ddayBadge`(D+N 부호) 폐기 → **`expiryBadge`**(만료 지남="만료됨" 빨강 / ≤30 "D-N" / 31~90 차분 / >90 배지 생략)·**`collectBadge`**(기한 지남="기한 지남" 빨강 / 그 외 "수금 D-N"). 앱 `docExpired`·`ddayText(collectDday/statusOverdue)` 규칙과 일치. 적용: `/w/home`·`/w/documents`(expiryBadge)·`/w/ledger`(collectBadge)·`/s`(만료됨 i18n·원거리 생략, `docExpired` 6언어 신설).
- **검증**: `/w/documents` — 만료 지남 "만료됨"(빨강)·건강진단서 "D-12"(임박)·먼 만료 배지 생략. `/w/ledger` — 기한 지난 미수 "기한 지남".

## 백엔드 요청사항 (서명 이미지 실렌더용 — app/backend 범위)
- 공개 확인서 API(`GET /public/confirmations/:token`)에 **서명 이미지 접근 수단** 추가 필요. 택1:
  1. 응답에 `signImageUrl`(공개 서명 이미지 라우트, 예 `/api/public/confirmations/:token/signature.png`) 추가, 또는
  2. base64 인라인(`signImageDataUrl`) — PDF 임베드에 이미 로드하므로 재사용 가능.
- 접근 가능해지면 웹 도장 박스 자리에 실제 서명 획을 렌더(백엔드 무수정 상태로도 `.seal` 인영으로 우선 개선 완료).
