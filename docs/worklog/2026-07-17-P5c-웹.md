## 무엇을 / 왜 / 어떻게

P5a 백엔드 3종 API를 "작업온" 사업장 웹(`/biz/*`)에 화면으로 노출. 모든 조회는 `useBiz().business.id` 로 `businessId` 스코프 부여(다중 소유 시 BusinessSwitcher 전환 전파), 이름은 서버 마스킹본 그대로 표시, 주민번호 비수집 원칙 UI 반영.

### 사이드바 메뉴 3개 추가 (`app/biz/layout.tsx`)
- NAV 배열에 `출역 현황`(/biz/attendance, Clock)·`현장 인건비`(/biz/site-costs, Coins)·`지급명세서`(/biz/wage-statement, Copy) 삽입 → 데스크톱 사이드바 + 모바일 상단 가로 nav 동시 반영(기존 `.biz-topbar-nav` 재사용).

### 1. 출역 현황판
- **홈 요약 카드** (`app/biz/inbox/page.tsx`): 상단에 `GET /biz/today-attendance` 요약(오늘 인원/출근/완료/미출근 4지표 타일) + `/biz/attendance` 링크. 공유 컴포넌트 `AttendanceStats`.
- **상세 페이지** (`app/biz/attendance/page.tsx` + `shared.tsx`): 현장별 카드 → 표(작업자·상태 배지[예정/수락/작업중/완료/취소]·시작~종료 시각·컨디션 배지). **30초 자동 갱신**(setInterval)·수동 새로고침(마지막 갱신 시각 표시). 페이지 라우트 규약상 임의 export 불가 → 타입·`AttendanceStats`·상태 메타는 `shared.tsx` 분리(inbox·상세 공유).

### 2. 현장별 인건비 (`app/biz/site-costs/page.tsx`)
- 월 범위 선택(시작/종료 `MonthPicker` ±1개월, 클라이언트에서 from>to·12개월 초과 사전 차단) → `GET /biz/site-costs?from=&to=&businessId=`.
- 전체 총계 카드(현장수·확인서수·연인원·금액) + **현장별 접이식 카드**(기본 전부 펼침): 작업자 행(연인원·공수·금액), 팀 확인서는 **팀 N명 배지**, 소계 행. 금액 내림차순(서버 정렬 유지).
- **PDF 다운로드**: `authedBlob('/biz/site-costs/pdf?...')` → `URL.createObjectURL` 새 탭(발주처 제출용).

### 3. 지급명세서 도우미 (`app/biz/wage-statement/page.tsx`)
- 월 선택(`MonthNav`) → `GET /biz/wage-statement?month=&businessId=`.
- **소득 유형 토글**(사업소득 3.3% ↔ 일용근로소득): 선택에 따라 작업자별·합계의 소득세·지방세·차인지급액 재계산 표시.
- **복사 버튼**: `copyText`(홈택스 입력용, 주민번호 비수집 안내 포함) → 클립보드(폴백 prompt) + 토스트.
- **월 마감 버튼**: `POST /biz/wage-statement/mark`(멱등) → 성공 시 `마감됨` 배지·버튼 `마감 완료` disabled, 마감 후에도 데이터 유지.
- **안내 노트 필수 표시**: `hometaxNote`(주민번호 비수집 경고 배너) + `notes`(5줄, "세무 상담 아님·세무사 확인 권장") 카드.
