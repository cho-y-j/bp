# 캘린더 미수/입금 분리 표기 (2026-07-18)

> 홈·캘린더 위계 배치 worklog의 "미해결/참고" 잔여 항목 해소: 캘린더 월 그리드·일별 장부·월 상단 합계가 청구 합계(billed)만 표시하던 것을 **정산 상태 반영(미수 주황 / 입금 초록)** 으로 확장. backend는 additive(기존 필드·응답 무변경), app은 파싱·표시 추가. web 무수정.

## 무엇을 / 왜 / 어떻게

### 1. backend (additive) — 확인서 월 조회에 정산 상태 부여
- 대상 엔드포인트: `GET /confirmations?month=YYYY-MM` (`confirmations.service.ts` `list()`), 캘린더 월 그리드·일별 뷰 원천.
- 연결된 1:1 ledger entry(`include: { ledgerEntry: true }`)를 기준으로 각 확인서에 `settlement` 추가:
  - `settlement: { paidAmount, outstandingAmount, status } | null`
  - `status`: `PAID`(미수 0·완납) / `PARTIAL`(일부 입금) / `UNPAID`(입금 0 — 장부 OVERDUE 도 색은 미수로 축약).
  - 순수 함수 `settlement.util.ts` `computeSettlement()`(장부 `computeOutstanding` 재사용) — 단위 테스트 대상.
- **파생(팀원) 이중 계상 없음**: 확인서 소유자의 목록에는 소유자 본인 확인서만 나온다. 반장 팀 확인서의 연결 entry 는 팀 합계 1건이고, 팀원 몫(derived=true)은 각 팀원 본인 장부의 별도 entry(소유자 목록엔 없음) → income-report 배제 규칙과 동일 원리로 자연 배제.
- 응답 top-level·byDate 에 additive 합계 추가(기존 `totalAmount`=billed 무변경):
  - top: `totalPaid`, `totalOutstanding`
  - byDate[]: `paidAmount`, `outstandingAmount`
- **ledger entry 생성 시점 = 확인서 생성(DRAFT)**: 현행 스키마상 확인서 생성 시 1:1 entry 가 항상 함께 생성된다(서명 시점 아님). 따라서 DRAFT/SENT 확인서도 settlement 가 존재(전액 미수 UNPAID). entry 가 없는 방어적 케이스만 null(주석 명시). 이 덕에 캘린더 미수/입금 총계가 홈 히어로(ledger summary)와 **정의·수치 일치**.

### 2. app — 모델 파싱 + 캘린더 표시(app/lib/features/calendar/calendar_screen.dart)
- 모델(`models.dart`): `Settlement` 신규, `Confirmation.settlement`·`isFullyPaid`, `DayAggregate.paidAmount/outstandingAmount/fullyPaid`, `ConfirmationList.totalPaid/totalOutstanding`.
- **날짜 칸 = 구글 캘린더 월뷰식 일감 미니 라인**(`_DayCell` 전면 재설계): 그날 확인서를 **최대 3줄**로 표시. 각 줄 = 현장명(1줄 말줄임, 9px) + 정산 상태 옅은 칩색 — **입금완료(SIGNED+완납) 초록 / 미수(SIGNED+잔존) 주황 / 초안·전송(DRAFT·SENT) 중립 회색**. 3건 초과분은 마지막 줄 `+N`. 금액은 칸에서 생략(무슨 일을 했는지 우선 — 합계·건별 금액은 탭 펼침 뷰). 셀 높이 `childAspectRatio 0.68→0.60`(6주 달도 한 화면).
- 날짜 탭 장부 뷰(`_DayLedger`/`_DayLedgerRow`): 그날 **전건** 리스트(단건 제한 없음, `where(date).toList()` 전부 렌더) — 현장·상대·시간·금액·상태 배지. 항목 금액 색(완납 초록 / 미수 주황) + **PARTIAL 이면 '입금 N원' 보조 표기 한 줄**(기존 `ledgerDeposited` 재사용). 날짜 헤더 합계 = 그날 합계(2건 이상이면 합산), 전건 완납이면 초록.
- 월 상단 히어로: "받을 돈" 숫자를 billed(`totalAmount`) → **미수 합(`totalOutstanding`)** 으로 교체 → 홈 히어로와 의미·수치 일치.
- 홈 "오늘 일정": 기존 `for (final conf in todays)` 로 오늘 확인서 **전건** 카드 렌더 — 코드상 이미 다건 지원(변경 없음, 확인만).

## 미해결 / 참고
- **팀원(파생 소득) 캘린더**: 팀원이 받는 파생 장부 항목은 소유 확인서가 없어 캘린더(확인서 기반)에 표시되지 않는다. 이 경우에만 캘린더 미수 총계 < 홈 히어로(ledger summary, 파생 포함)로 벌어질 수 있음. 시드(본인 확인서만)에선 정확 일치. 파생 소득을 캘린더에 노출하려면 별도 설계 필요(백로그).
