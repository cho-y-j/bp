## 1. 작업 1 — 글리프 깨짐 수정 (치명 · ★1)

### 원인 규명 (재현으로 확정)
- 증상(감사): NanumGothic 임베드본에서 **숫자 0~9 전부 + 한글 다수 글리프 미렌더**. "1,050,000원" → "` , , 원`", "대한건설(주)" → "한건설". poppler·Apple CoreGraphics 두 엔진 동일 재현.
- 진단: 폰트는 `emb=yes` 이고 `pdftotext` 텍스트 레이어는 정상 → **글리프 매핑 파손**. 코드가 `pdf.embedFont(bytes, { subset: true })` 사용.
- **최소 재현**: 동일 폰트를 `subset:true` / `subset:false` 로 각각 임베드해 같은 문자열을 그려 `pdftoppm` 렌더 비교.
  - `subset:true` → 숫자 전멸·한글 다수 누락(감사 증상과 동일 재현).
  - `subset:false` → "작업확인서 1,050,000원 2026-07-06 대한건설(주) 김현수 0123456789" **전부 정상 렌더**.
- 결론: `@pdf-lib/fontkit@1.1.1` 의 **서브셋(subset) 매핑 버그**. glyf/CIDToGIDMap 서브셋 생성이 깨져 CID→글리프 변환 실패. 뷰어 문제 아님.

### 수정
- 폰트 임베드를 **`subset: false`(전체 임베드)** 로 전환 → 글리프 정상.
- 타이포 위계(굵은 헤더·합계 강조)를 위해 **NanumGothic Bold 를 로컬 OFL 재확보**해 함께 임베드.
  - 출처: 로컬 캐시 `/Users/jojo/pro/mybot/backend/app/reports/fonts/NanumGothicBold.ttf`(동일 OFL NanumGothic 패밀리, 외부 다운로드 없음) → `backend/assets/fonts/NanumGothic-Bold.ttf` 로 복사. (Regular 옆 `OFL.txt` 라이선스 유지)
- `embedFonts()` 로 Regular/Bold 를 캐시·전체 임베드. 코드 주석에 서브셋 금지 사유 명시(회귀 방지).

### 검증 (pdftoppm 이미지 렌더 → 눈검수, 필수 기준 충족)
- `pdffonts` 재확인: 두 폰트 모두 `emb=yes / sub=no / uni=yes`(전체 임베드 확정).
- 전후 이미지:
  - Before(깨짐, 감사): `web/screenshots-audit/pdf/01-confirmation-page-1.png`, `02-statement-page-1.png` 등 6종 — 숫자·한글 누락.
  - After(정상): `web/screenshots-audit/pdf-fixed/01-confirmation-fixed.png` ~ `06-site-costs-fixed.png` — 숫자·한글·서명 이미지·표 정렬 **전부 정상**.

## 2. 작업 2 — 문서 리디자인 (DESIGN-IMPROVE P0-2)

공통 헬퍼로 6종 서식을 통일(확인서·명세서·안전리포트·계약서 4종 집중, 소득리포트·현장인건비도 동일 골격 적용):
- **문서 헤더**: 문서명(24pt 굵게) + 부제 + 우측 발행처(작업온·workon)·발행일·상태. 하단 네이비 괘선 + 좌측 오렌지 포인트(절제).
- **명확한 괘선 표**: 헤더 밴드(종이톤 `#F7F6F3` 채움) + 행 구분선 + 짝수행 줄무늬. 금액 **우측정렬**(tabular). 미수=`#C2410C`·입금=`#15803D` 색 병행.
- **합계 강조 박스**: 옅은 오렌지 배경 + 오렌지 테두리 + 큰 굵은 금액(확인서 "청구 합계", 명세서·현장인건비 "총계" 밴드).
- **서명 도장 박스**: 이중 테두리(직인 느낌) 안에 **서명 이미지** + 아래 **서명자명 + 서명일시**. 확인서 1개·계약서 2개(사업주/근로자).
- **푸터**: 전 페이지 하단 "작업온에서 발행 · workon" + (다중페이지) 쪽번호.
- 여백(MARGIN 48)·색 토큰(DESIGN-UI 팔레트)·타이포 위계 통일. 과한 장식 배제 — 격식·명료 우선.
- 리디자인 중 발견·수정한 레이아웃 버그: 명세서 입금/미수 컬럼 간격 부족으로 금액 겹침 → 컬럼 재배치. 현장인건비 헤더 우측 4번째 줄(기간)이 괘선과 겹침 → 헤더 괘선을 우측 블록 아래로 내리도록 일반화.
