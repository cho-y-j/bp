# S4a 앱 코어 (Flutter, 작업자 관점) — 2026-07-11

## 구현 범위
`flutter create`(org kr.workon / project workon / iOS·Android) 위에 작업자 코어 앱을 구현했다.
상태관리 **Riverpod**, 라우팅 **go_router**, HTTP **dio**, 보안저장 **flutter_secure_storage**,
공유 **share_plus**, 파일 **path_provider + open_filex**, 숫자/날짜 **intl**.
백엔드(S2, 임시 pg **5435** / API **3030**)와 실연동해 iOS 시뮬레이터(iOS 26.5)에서 E2E 검증까지 마쳤다.

### 코어 인프라
- `lib/core/api_client.dart`: dio 래퍼 — **`{data,error}` 봉투 언래핑**, JWT 자동 주입, **401 → onUnauthorized(강제 로그아웃)**,
  인증 blob(`getBytes`, responseType bytes)로 PDF 수신. BASE_URL 은 **dart-define**(기본 `http://localhost:3030/api`).
- `lib/core/token_store.dart`: JWT 를 iOS Keychain/Android Keystore 에 보관(flutter_secure_storage).
- `lib/core/amount_calc.dart`: 백엔드 `amount.util.ts` 를 미러한 클라이언트 금액 **미리보기** 계산(저장은 서버값 사용).
- `lib/core/format.dart`: 천단위 콤마·요일·D-day·월/일 파라미터 헬퍼.
- `lib/providers/auth.dart`, `data.dart`: apiClient/auth 컨트롤러 + FutureProvider(family) — 확인서·장부요약·회사별·항목·만료서류·연결.

## 백엔드 변경 (S4a 필수 추가 — 최소·비파괴)
- 작업자 장부 상세에서 **개별 항목 id 로 부분입금**을 기록하려면 항목 id가 필요한데 기존 API에 없었음
  (`by-company` 는 집계만, `confirmations` 는 ledgerId 미노출). → **`GET /ledger/entries?month=&businessId=` 추가**:
  `LedgerService.entries()` + 컨트롤러 라우트. 반환은 LedgerDto + 해소된 companyName/siteName/date.
  변경 파일: `backend/src/ledger/ledger.service.ts`, `backend/src/ledger/ledger.controller.ts`.
- 기존 로직 무수정(추가만). `nest build` 통과, 기존 **ledger 단위 12/12 통과**. curl 로 신규 엔드포인트 동작 확인.
- (백로그) 신규 엔드포인트 e2e 테스트는 미작성 — S4b/백엔드 회귀 시 추가 권장.

- **인증 PDF**: 명세서/확인서 PDF 는 Authorization 헤더 blob(`ApiClient.getBytes`) 패턴 이미 구현 — 재사용.
- **datetime→ISO**: 작업 지시 `scheduledAt` 은 ISO8601(로컬 → toIso8601String).
- **개발용 ATS**: `ios/Runner/Info.plist` 에 `NSAllowsArbitraryLoads=true`(로컬 http 접근용) — **운영 빌드 전 제거/HTTPS 전환**.
