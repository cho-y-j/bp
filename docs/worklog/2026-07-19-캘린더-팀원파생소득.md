# 캘린더 팀원 파생 소득(teamShares) 표시 (2026-07-19)

> 직전 worklog(2026-07-18 캘린더 미수/입금 분리)의 "미해결 — 팀원 파생 소득" 해소.
> 팀원(반장 팀 확인서에 포함된 가입·연결 작업자)은 서명 시 파생 장부 항목(본인 몫)만 생기고
> 본인 확인서가 없어 캘린더에 그날이 비어 보이고, 캘린더 미수합 < 홈 히어로(파생 포함)로 어긋났다.
> backend additive + app 표시로 해소. web 무수정.

## 무엇을 / 왜 / 어떻게

### 1. backend (additive) — `GET /confirmations?month=` 에 teamShares 추가
- `confirmations.service.ts` `list()` 확장. 원천: 본인(profileId) 소유의 `derived=true` ledger entry
  (반장 팀 확인서 서명 시 생성). 본인 소유 확인서가 아니므로 `items` 에는 절대 없다
  (`confirmationId=null`, `sourceConfirmationId` 로만 원 확인서와 연결) → items 와 겹칠 수 없어 이중 계상 없음.
- teamShares 각 항목: `{ id(파생 entry id), date(원 확인서 작업일), site(원 확인서), teamLeaderName(파생
  생성 시 스냅샷된 반장 이름=counterpartyName), amount(본인 몫), settlement:{paidAmount,outstandingAmount,status} }`.
- **집계 재일치**: `count`·`totalAmount`·`totalPaid`·`totalOutstanding`·`byDate`(count/totalAmount/paid/outstanding)
  에 teamShares 를 함께 합산. 홈 히어로(ledger summary)는 파생 entry 를 포함해 집계하므로, 캘린더 총계도
  teamShares 포함 → **totalOutstanding/totalPaid/totalBilled 3필드가 ledger summary 와 정확 일치**.
- month 미지정 시엔 전체 파생(sourceConfirmationId 존재) 집계.

### 2. app — 캘린더에 teamShares 반영 (`calendar_screen.dart`, `models.dart`)
- 모델: `TeamShare` 신규 + `ConfirmationList.teamShares`(누락 시 빈 리스트 안전 파싱).
- **날짜 칸 미니 라인**: 확인서 라인과 동일 규칙(정산 상태 색)이되 **"팀·" 프리픽스**로 구분
  (`팀·판교 팀 현장`). 팀 작업은 서명 후 생성이므로 초안 없음 → 완납 초록/미수 주황. 확인서+팀 작업을
  합쳐 3줄 제한·+N 규칙에 함께 포함(`_MiniLine` 공통 표현으로 통합).
- **날짜 탭 패널**: 읽기 전용 카드 `_DayTeamShareRow` — "OOO 반장 팀 작업" 라벨(기존 `ledgerTeamDerived`
  재사용) + 본인 몫 금액(정산 상태 색) + PARTIAL '입금 N원' 보조줄(기존 `ledgerDeposited`) + 팀 배지 + 현장.
  **탭해도 확인서 상세로 이동하지 않음**(비인터랙티브 — 본인 소유 문서 아님). 은은한 배경 tint 로 파생 구분.
- 그날 합계·전액완납 판정에 팀 작업 몫 포함. 월 상단 "받을 돈"(totalOutstanding)·"작업 N건"(count)은
  서버가 이미 teamShares 포함해 내려주므로 그대로 신뢰(앱 추가 계산 없음).
- **주간(리스트) 뷰**에도 동일 반영(같은 버그가 주간 뷰에서 재현되지 않도록) — 읽기 전용 카드 재사용.
- 홈 "오늘 일정"에는 넣지 않음(일정 아님). 홈은 `list.items` 만 순회 → 팀 작업 자동 제외(코드 변경 없음).

### ARB
- **신규 키 없음.** 기존 `ledgerTeamBadge`("팀")·`ledgerTeamDerived`("{boss} 반장 팀 작업")가 요구 문구와
  정확히 일치해 재사용(6언어 이미 존재·패리티 무변경). 직전 settlement 작업과 동일한 "무신설" 판단.

## 미해결 / 참고
- 미가입(수기) 팀원은 파생 entry 자체가 생성되지 않으므로(설계상 가입·연결 팀원만) 캘린더에도 표시되지 않음 — 스펙 준수.
- teamShares 카드는 비인터랙티브(탭 무동작). 장부 화면으로의 딥링크 이동은 탭 네비 구조상 추가 설계 필요(백로그, UX 자연스러움 우선으로 무동작 선택).
