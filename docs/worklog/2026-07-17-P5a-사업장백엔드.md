### 1. 현장별 인건비 집계 — `GET /biz/site-costs?from=&to=&businessId?=` (+ `/pdf`)
- 사업장 대상 **SIGNED 확인서**(confirmation.businessId ∈ scope)를 **현장명(site)별**로 집계. 순수 함수 `src/biz/site-costs.util.ts (aggregateSiteCosts)`:
  - 일반 확인서: 작업자(profileId)별 1행 — 이름 마스킹(maskName), **연인원(man-days)**·공수·금액·건수 합산.
  - 팀 확인서(teamEntries 有): 반장 기준 1행, `isTeam=true`·`teamMemberCount`(팀원 수), 팀 합계 금액.
  - 현장 소계(=행 합)·전체 총계(=현장 소계 합). **정합성 검증 내장**(소계 합 ≠ 총계면 예외 → 회귀 방지).
  - 정렬: 현장·행 모두 금액 내림차순.
- `workUnitsOf(confirmation)`(biz.service): 팀=Σ공수, GONGSU=기본수량, DAILY=일수(quantity), 그 외(HOURLY/PER_CASE/…)=1. days=연인원, gongsu=공수.
- 기간 **최대 12개월**(`resolveSiteRange`: from>to → 400 INVALID_RANGE, 12개월 초과 → 400 RANGE_TOO_LONG, 형식 오류 → 400 INVALID_MONTH).
- PDF `PdfService.renderSiteCostsPdf` — **사업장 상호·기간 헤더** + 현장별 표(작업자/팀·일수·공수·금액) + 현장 소계 + 전체 총계 + 마스킹 안내(발주처 제출용, 나눔고딕 A4 페이지브레이크). 인증 blob `Content-Disposition inline`.
- **팀 파생 중복 방지**: 사업장 입장 확인서는 "반장 팀 확인서 1장"만 존재(팀원 파생 ledger 는 반장 개인 장부에만 생성). 사업장 대상 confirmations 만 집계하므로 이중 합산 없음.

### 2. 일용근로소득 지급명세서 도우미 — `GET /biz/wage-statement?month=&businessId?=` + `POST /biz/wage-statement/mark`
- **지급(payments의 paidAt) 기준** 월별 작업자별 집계: 지급 대상 = 사업장 대상 SIGNED 확인서의 ledger_entry 중 당월 paidAt 입금이 있는 항목(부분입금 여러 건은 확인서 단위로 당월 합산). 지급총액·지급 건수·근로일수(확인서 일수, 공수 포함).
- **소득 유형별 세액 동시 산출**(순수 함수 `src/biz/wage-statement.util.ts`, 상수+주석으로 기준 명시 — 2026 기준):
  - **business3_3**: 소득세 지급액×3%, 지방소득세 소득세×10%(=0.3%), 원단위 절사. 차인지급액 = 지급액−원천징수.
  - **dailyWage(일 단위)**: 1일 근로소득공제 150,000 → 과세표준×6% 산출세액 → 근로소득세액공제 55% 감면(결정세액=산출×45%, 즉 과세표준×**2.7%**) → **1일 세액 1,000원 미만 소액부징수 0** → 일수만큼 합산. 지방소득세 소득세×10%. **일급 = 지급액 ÷ 일수(확인서 일수/공수)**.
  - 응답 `notes`(5줄): "세율은 2026 기준 참고값·세법 개정 확인·세무 전문가 확인 권장", "**세무 상담이 아닙니다**", "주민번호 비수집" 명시.
- **주민번호 비수집** — 응답 `hometaxNote`("홈택스에서 직접 입력") + copyText 상단 안내.
- **홈택스 입력용 복사 텍스트**(`copyText`) — 작업자별 지급총액·일수·소득 유형별 세액·차인지급.
- `POST /biz/wage-statement/mark` — 월 마감 표시(`businesses.wageMarkedMonths` push). **멱등**(이미 있으면 추가 없이 `alreadyMarked:true`). 마감 후에도 **데이터 계속 조회 가능**, GET 응답 `marked` 로 표시(스코프 내 전 사업장이 마감했을 때 true).

### 3. 오늘의 출역 현황판 — `GET /biz/today-attendance?businessId?=`
- **오늘(KST)** scheduledAt 인 사업장 jobs 를 **현장(site)별 그룹**. 각 작업자 상태(work_logs 최신 1건 조인): `SCHEDULED`(예정)/`ACCEPTED`(수락)/`STARTED`(시작·startedAt·condition)/`DONE`(완료·finishedAt)/`CANCELLED`. 이름 마스킹.
- 현장별·전체 **인원 요약**: total(취소 제외)/attended(시작+완료)/completed(완료)/absent(미출근).

## P5b(앱)·P5c(웹)에 넘길 API 계약
- `GET /biz/site-costs?from=YYYY-MM&to=YYYY-MM&businessId?=` → `{ range, businessName, sites:[{ site, entries:[{ workerName(마스킹), isTeam, teamMemberCount, days, gongsu, amount, entryCount }], subtotalAmount, subtotalDays, subtotalGongsu, workerCount }], totals:{ totalAmount, totalDays, totalGongsu, siteCount, entryCount } }`. 기간 최대 12개월. `GET /biz/site-costs/pdf?...` → PDF blob(발주처 제출용).
- `GET /biz/wage-statement?month=YYYY-MM&businessId?=` → `{ month, businessName, marked, workers:[{ workerName, paidTotal, paymentCount, workDays, business3_3:{incomeTax,localTax,totalTax,netPay}, dailyWage:{...} }], totals:{ workerCount, paidTotal, paymentCount, business3_3, dailyWage }, notes:[], hometaxNote, copyText }`. `POST /biz/wage-statement/mark { month, businessId? }` → `{ businessId, month, marked:true, alreadyMarked }`(멱등).
- `GET /biz/today-attendance?businessId?=` → `{ date(YYYY-MM-DD KST), sites:[{ site, workers:[{ jobId, workerName, status:SCHEDULED|ACCEPTED|STARTED|DONE|CANCELLED, scheduledAt(HH:mm), startedAt, finishedAt, condition }], summary:{total,attended,completed,absent} }], summary:{...} }`.
- 공통: 모든 `businessId?` 는 additive(미지정=소유 전체, 지정=해당 1건, 미소유=빈 결과). 인증 필수(미인증 401). 이름은 서버에서 마스킹 완료본 제공. **주민번호는 클라이언트도 수집 금지** — wage-statement 는 홈택스 직접 입력 안내.

## 미완 / 다음 배치(P5 통합) 주의
- **부분입금 + 일용 세액**: wage-statement 는 확인서(지급 건) 단위로 당월 지급액을 합산해 일급=지급액÷확인서일수 로 산정. 한 확인서가 여러 달에 걸쳐 부분입금되면 일급이 실제와 달라질 수 있음(문서화). 실무상 확인서 단위 완납이 일반적.
- **팀 반장 소득 유형**: 팀 확인서는 사업장이 반장에게 팀 전체 금액을 지급 → 지급명세서에 반장 1인으로 집계(팀원 재분배는 반장 개인 장부의 파생 항목). 반장은 보통 3.3% 사업소득으로 처리(둘 다 산출 제공, 사용자 선택).
- **today-attendance**: 오늘 scheduledAt 기준. 여러 날 예약/미래 예약은 제외. work_logs 최신 1건 기준 상태.
