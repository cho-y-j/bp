2. **labor-contracts 모듈**(`src/labor-contracts/`): 3개 컨트롤러 — `biz/contracts`(사업장), `contracts`(작업자), `public/contracts`(외부 @Public). 서비스 + 매퍼 + DTO 3종.
3. **작성/수정/삭제**(사업장 소유자만, `ownedContractOrThrow`): create 시 사업장 소유·작업자(전화검색 동의 or ACCEPTED 연결) 검증. update/delete 는 DRAFT + 사업장 미서명일 때만(409 NOT_EDITABLE/NOT_DELETABLE).
4. **양측 서명 흐름**:
   - `POST /biz/contracts/:id/sign-employer`: 사업장(대표) 서명(DRAFT 에서만, 상태 유지). PNG 검증(1MB·시그니처) 재사용. TOCTOU 방지 조건부 updateMany.
   - `POST /biz/contracts/:id/send`: 사업장 서명 필수 선행(없으면 409 EMPLOYER_SIGNATURE_REQUIRED) → SENT. 연결 작업자 알림 or 수기 링크(알림톡 CONFIRMATION_SIGN 재사용). `url=/lc/{token}`.
   - 작업자 서명(`POST /public/contracts/:token/sign` 외부, `POST /contracts/:id/sign` 앱내): SENT 만, **원자적 SIGNED 전이**(updateMany where status=SENT & workerSignedAt=null; count=0 → 409 ALREADY_SIGNED = 재서명 차단), 양측 알림.
5. **작업자 "내 계약서"**: `GET /contracts` = workerProfileId 연결 + status SENT|SIGNED → 서류 지갑 자동 연결(별도 조인 불필요). 앱내 서명·PDF 열람.
6. **PDF**(`PdfService.renderLaborContractPdf`): 조항 번호 나열(당사자·기간·근무장소·업무·근로시간·임금·지급·수당·4대보험·특약) + **정본 안내("본 계약서의 정본은 한국어본입니다")** + 하단 **사업주/근로자 2개 서명 박스**(양측 서명 이미지·서명자명·서명일시).
7. **권한 격리**: 사업장 계약서는 소유자만(남 404), 작업자 계약서는 연결 작업자만(남 404). 공개는 토큰만.

## 미완 / 다음 배치(P2c TBM) 주의사항
- **revoke API 미노출**: `revokedAt` 컬럼·공개 열람 차단 로직은 있으나 무효화 엔드포인트(예: `POST /biz/contracts/:id/revoke`)는 미구현(확인서와 동일한 상태 — 필요 시 공통으로).
- **SENT 후 수정/재발행 없음**: 전송 후 내용 변경은 새 계약서 재작성 흐름(DRAFT 만 수정·삭제). 계약 변경(갱신) 이력 모델은 향후 과제.
- **P2c(TBM) 주의**: ①서명/확인 탭 기록이 필요하면 이번 labor-contracts 의 **조건부 updateMany 원자 전이 패턴**(TOCTOU 409) 재사용. ②safety_logs 는 append-only(updatedAt 없음) — TBM 기록도 같은 원칙 유지, 참석자 확인은 ackAt 최초 1회 UPDATE 패턴 참고. ③사업장 소유 검증은 `ownedBusinessOrThrow`(labor-contracts.service) 패턴 재사용 가능. ④알림 유형이 CONFIRMATION 으로 뭉치고 있음 — TBM 은 필요 시 NotificationType enum 추가(additive 마이그레이션). ⑤안전 리포트 PDF 에 TBM 섹션은 `renderSafetyReportPdf` 확장이 자연스러움.
- 웹 `/lc` 는 공개 열람·서명 페이지만 존재(사업장 웹에서 계약서 발행·관리 UI 는 미구현 — 현재 앱 전용, 필요 시 P2 통합에서).
