# 작업온 차별화 전략 (2026-07-11 시장 리서치 기반)

> 근거: 경쟁 앱 웹 리서치 (일용이/공수계산기/장비박사/장비불러/화물고수/가다/김반장/세모장부/청구스/안전지키미 등)

## 시장 구조: 5개 사일로 — end-to-end는 공백
① 공수계산 앱(개인 가계부) ② 장비 배차 플랫폼 ③ 고용주 노무 SaaS ④ 회계·미수금 앱 ⑤ 안전관리 앱
→ **"작업확인서 전자서명→장부 자동→정산·미수금→안전 증빙"을 양쪽(작업자↔소사업장)으로 잇는 앱은 없음.**
→ 실재 페인포인트: 장비 대금 수개월 체불(언론 보도), 중대재해처벌법 안전 증빙 의무.

## 작업온만의 차별점 (구현 완료)
1. **쌍방 서명 작업확인서** — 경쟁 앱은 "내 기록"(일방) 또는 "근로계약서"만. 우리는 작업량·단가를 상대가 서명으로 합의 → 체불 분쟁의 증거
2. **미가입 상대도 카톡 링크로 서명** — 설치 없이. 5개 사일로 어디에도 없는 콜드스타트 해법이자 성장 엔진
3. **확인서→장부→정산 자동 체인** — 청구스(회계)와 공수앱(기록)이 못 잇는 연결. 쓰면 장부가 저절로 생김
4. **안전 증빙 자동 축적** — 폭염 알림·서류 유효확인·컨디션 체크가 자동 기록 → 소사업장의 중대재해법 대응 무료 도구 (안전 사일로 흡수)
5. **서류 지갑 + 마스킹 + 만료 관리** — 장비 기사 특화 강화
6. **업종 포괄 + 무료** — 장비/일용/기술자 모두, 양면 무료

## 추가 로드맵 (우선순위)

### P1 — 출시 전·직후 (현장 밀착 편의)
- **공수(품) 단위 지원**: 일용직 표준 언어 — 0.5공수/1.5공수 입력 (quantity 소수 허용 + "공수" 단가유형 라벨)
- **홈 화면 위젯**: 오늘 일정 + 이번 달 미수금 (앱 안 열고 확인)
- **오프라인 대응**: 현장 신호 불량 시 확인서 임시저장 → 연결 시 자동 전송
- **카카오 로그인 실연동** + **알림톡**(미가입 상대 전달·수신확인 — 카톡 공유보다 도달 강력)
- 세금계산서 1단계: 정산 확정 → 홈택스 입력용 데이터 자동 정리
- **다국어 (P2→P1 승격, 2026-07-11 사용자 강조 "대박")**: i18n 기반 구조를 지금 구축(후행 이식은 비용 큼)
  + **지원 언어 확정 (2026-07-11 리서치 기반, worklog/2026-07-11-외국인노동자-조사.md)**:
    한국어(기본) + 번역 5종 = **중국어 간체, 러시아어, 베트남어, 네팔어, 영어**
    - 중국어: 건설현장 외국인의 89.6%가 중국어권(조선족 83.7%+본토 5.9%)
    - 러시아어: 우즈벡·고려인·카자흐·키르기스 **중앙아시아권 통합 커버** (우즈베크어 별도 불필요 — 러시아어 병용)
    - 베트남어: 비한국어권 최대 취업자(14.9만, 2위)
    - 네팔어: 등록외국인 4.8%, 일용직·제조 확장 대비
    - 영어: 필리핀·스리랑카 및 범용 폴백
  + 2차 확장: 크메르어(캄보디아)·인도네시아어·태국어·미얀마어·몽골어
  + 핵심: 앱 UI + **확인서 외부 열람 페이지**(외국인 작업자가 자기 언어로 읽고 서명)
  + 확인서 PDF는 한국어 기본 + 이중 언어 옵션

### P2 — 성장기 (사일로 흡수)
- **반장(팀) 기능**: 반장이 팀원 여러 명 공수 일괄 기록·일괄 확인서 (일용직 실사용 구조)
- **표준근로계약서 전자서명**: 김반장(유료 SaaS) 영역을 무료로 흡수
- **간편 TBM/위험성평가 기록**: 안전 앱 영역 흡수 — 안전 리포트에 통합
- **연간 소득 리포트**: 종합소득세 대비 (삼쩜삼 제휴 여지)

### P3-신규 (2026-07-11 확정, 사용자 승인 "바로 착수") — 돈 받기·신뢰·확산
1. **수금 독촉 자동화**: 수금 D+7/D+30에 앱이 작업자 대신 사업장에 점잖은 알림톡/푸시("명세서+계좌 안내") — 작업자는 버튼 1개(자동 독촉 ON). 발송 이력 기록
2. **지급 평판 배지**: 정산 데이터로 사업장 "평균 지급 소요일" 자동 산출 → 우수 지급처 배지(데이터 기반, 주관 리뷰 없음). 검색·프로필에 노출
3. **QR 명함**: 작업자 공개 프로필 QR(이름·업종·서류 유효 배지·연결 버튼 웹 페이지) — 오프라인 확산 루프
- 기타 후보(백로그): 분쟁 증거 패키지, 지급보증 체크, 음성 확인서, 타임스탬프 카메라, 우천 일정, 장비 차계부, 5월 환급

### 구인구직 매칭 (2026-07-18 사용자 문의 → 보류 결정)
- **지금 안 만든다**: ①마켓플레이스 콜드스타트(공고 없는 판 = 이탈) — 현재 강점은 상대 미가입도 되는 단독 사용 가치 ②가다·일당백 등 기존 경쟁 치열 ③직업소개는 직업안정법 신고·허가 부담
- **나중에 붙일 때의 무기**: 작업 이력 + 지급 평판 배지 + 서류 유효성 = 신뢰 레이어. "검증된 매칭"으로 차별화 가능. 착수 트리거: 연결망 형성 후 "사람 구해줘" 수요가 실제 발생할 때
- **중간 단계(백로그, 추천)**: **빈 날 공유** — 작업자가 캘린더에 가능한 날 표시 → 이미 연결된 사업장만 보고 작업 요청. 기존 연결망 내 기능이라 콜드스타트·직업소개 이슈 없음, 달력 정체성과 부합

### P4 문자 연동 (2026-07-17 사용자 아이디어, 착수 확정)
1. **문자로 보내기 원탭**: 확인서 서명 링크·서류 묶음 링크 전송 시 문자 작성창 자동 오픈(상대 번호+본문 링크 미리 채움, 전송은 사용자) — iOS·Android, 정책 무риск
2. **빠른 전송**: 서류 지갑에 명함·사업자·통장 즐겨찾기 → 탭 1회로 문자 작성창
3. **통화 후 명함 보내기 (Android 전용)**: 통화 종료 감지 → **오버레이 팝업 대신 알림**("방금 통화한 번호로 명함 보낼까요?") → 탭 시 문자 작성창. 설정 기본 OFF+권한 고지 (자동 발송·오버레이는 구글 정책 리스크로 기각)
